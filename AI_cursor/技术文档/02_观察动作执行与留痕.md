# 02 · 观察、动作、执行与留痕
 
## 核心判断
 
AI 光标要能替用户操作电脑，必须形成稳定闭环：
 
```text
看见屏幕
→ 生成候选目标
→ 输出结构化动作
→ 安全评估
→ 可见执行
→ 观察结果
→ 写入事件日志
→ 继续 / 询问 / 恢复 / 完成
```
 
这个闭环比“让模型看截图点坐标”更重要。
 
---
 
## 多源屏幕感知
 
屏幕感知不能只依赖截图，也不能只依赖 DOM。更稳妥的方式是多源融合：
 
```text
Screenshot
+ OCR
+ DOM
+ Accessibility Tree
+ Window State
+ Cursor State
+ Clipboard
```
 
### Screenshot
 
适合：
 
- 跨软件统一观察
- 视觉布局判断
- 鼠标目标高亮
- 用户监督和回放
 
缺点：
 
- 结构化程度低
- 成本高
- 需要视觉模型或 OCR
 
### OCR
 
适合：
 
- 识别按钮文字
- 识别网页标题
- 识别表单标签
- 支持屏幕标号
 
### DOM
 
适合网页：
 
- 获取链接
- 获取按钮
- 获取输入框
- 获取表单结构
- 获取页面文本
 
缺点：
 
- 只适用于网页
- 有些网页结构混乱
- Shadow DOM / Canvas / 跨域 iframe 可能有问题
 
### Accessibility Tree
 
适合桌面和浏览器可访问性元素：
 
- 按钮
- 输入框
- 菜单
- 标签页
- 窗口控件
 
它比 OCR 更结构化，比 DOM 更跨应用。
 
### Window State
 
需要知道：
 
- 当前前台窗口
- 所有打开窗口
- 窗口位置和尺寸
- 浏览器标签页
- 分屏布局

## 目标候选生成
 
Observation 模块不应只把原始截图丢给模型，而应生成目标候选：
 
```json
[
  {
    "id": "element_1",
    "type": "input",
    "label": "搜索",
    "bounds": [120, 80, 620, 120],
    "source": "accessibility",
    "confidence": 0.92
  },
  {
    "id": "element_2",
    "type": "button",
    "label": "筛选",
    "bounds": [760, 210, 830, 248],
    "source": "ocr",
    "confidence": 0.81
  }
]
```
 
模型、规则或本地命令识别器再根据用户指令选择目标。
 
---
 
## 屏幕标号
 
屏幕标号是 MVP 的关键交互。
 
流程：
 
```text
观察屏幕
↓
生成可点击 / 可操作元素
↓
给元素编号
↓
在屏幕上 overlay 编号
↓
用户说“点 3”
↓
执行器点击 element_3
```
 
标号要支持：
 
- 点击
- 右键
- 拖拽
- 输入
- 滚动区域
- 标签页
- 窗口

## 动作协议核心原则
 
AI 不应直接输出：
 
```text
我去点一下右边的按钮
```
 
也不应直接输出裸坐标：
 
```json
{"x": 842, "y": 317}
```
 
而应输出结构化动作：
 
```json
{
  "action": "mouse.click",
  "target": {
    "type": "screen_element",
    "ref": "element_3",
    "description": "第一个搜索结果"
  },
  "button": "left",
  "safety": "safe",
  "expected_result": "打开搜索结果详情页"
}
```
 
执行器再把目标映射到真实坐标或系统 API。
 
---
 
## 动作协议分类
 
### Mouse Actions
 
```json
{"action": "mouse.move_to", "target": {"ref": "element_1"}, "duration_ms": 300}
```
 
```json
{"action": "mouse.click", "target": {"ref": "element_2"}, "button": "left", "click_count": 1}
```
 
```json
{"action": "mouse.drag", "source": {"ref": "tab_3"}, "destination": {"ref": "window_left"}, "duration_ms": 800}
```
 
```json
{"action": "mouse.scroll", "target": {"ref": "current_page"}, "direction": "down", "amount": "medium"}
```
 
### Keyboard Actions
 
```json
{"action": "keyboard.type", "text": "AI 光标产品文档"}
```
 
```json
{"action": "keyboard.hotkey", "keys": ["ctrl", "l"]}
```
 
```json
{"action": "keyboard.press", "key": "enter"}
```
 
### Browser Actions
 
浏览器动作可以由浏览器 API 执行，也可以由鼠标键盘模拟执行。
 
```json
{"action": "browser.new_tab"}
```
 
```json
{"action": "browser.open_url", "url": "https://example.com"}
```
 
```json
{"action": "browser.group_tabs", "tabs": ["tab_1", "tab_2", "tab_3"], "group_name": "求职资料"}
```
 
### Window Actions
 
```json
{"action": "window.arrange", "layout": "two_columns", "left": "browser_window_1", "right": "document_window_1"}
```
 
```json
{"action": "window.focus", "target": "browser_window_1"}
```
 
### Clipboard Actions
 
```json
{"action": "clipboard.copy_selection"}
```
 
```json
{"action": "clipboard.paste"}
```
 
### Control Actions
 
```json
{"action": "task.pause", "reason": "等待用户确认"}
```
 
```json
{"action": "user.ask", "question": "这里需要提交表单，要继续吗？", "options": ["继续", "取消", "我来接管"]}
```

## 动作对象结构
 
每个动作建议包含：
 
```json
{
  "id": "step_001",
  "task_id": "task_abc",
  "action": "mouse.click",
  "target": {},
  "params": {},
  "safety": "safe | confirmation_required | blocked",
  "expected_result": "",
  "fallback": "",
  "timeout_ms": 5000
}
```
 
---
 
## 安全等级
 
### safe
 
可以直接执行：
 
- 滚动
- 切标签
- 打开网页
- 点击普通链接
- 输入草稿
- 复制内容
 
### confirmation_required
 
必须确认：
 
- 提交
- 发送
- 删除
- 覆盖
- 授权
- 付款
- 发布
- 申请
 
### blocked
 
默认禁止：
 
- 绕过验证码
- 自动输入敏感密码
- 未确认付款
- 未确认删除大量文件
- 未确认发送外部消息
 
---
 
## 执行器职责
 
执行器不是简单调用系统 API，它还要负责：
 
1. 目标定位
2. 动作前高亮
3. 执行动画 / 光标移动
4. 执行后观察
5. 判断是否达到预期
6. 失败重试或询问用户
7. 写入日志
 
每个动作完成后返回：
 
```json
{
  "step_id": "step_001",
  "status": "success | failed | uncertain | interrupted",
  "observation": "页面已打开岗位详情",
  "screenshot_ref": "screen_001",
  "error": null,
  "next_required": "continue | observe | ask_user"
}
```

## 任务留痕
 
每个任务都应记录为事件流，而不是只保存最终结果。
 
### TaskRun
 
```json
{
  "task_id": "task_001",
  "goal": "帮我找 10 个适合我的岗位",
  "status": "running",
  "input_mode": "voice",
  "workspace": "browser_window_1"
}
```
 
### StepEvent
 
```json
{
  "event_id": "event_001",
  "task_id": "task_001",
  "type": "action_executed",
  "user_command": "打开第一个结果",
  "observation_ref": "obs_001",
  "action": {
    "action": "mouse.click",
    "target": "element_2"
  },
  "result": "success"
}
```
 
### UserCorrection
 
```json
{
  "type": "user_correction",
  "text": "不是这个，点右边那个",
  "previous_step": "event_001",
  "resolution": "target_changed"
}
```
 
---
 
## 为什么留痕重要
 
### 1. 安全复盘
 
用户可以知道 AI 做过什么。
 
### 2. 错误恢复
 
出错后可以回到上一步上下文。
 
### 3. 工作流沉淀
 
多次类似任务后，可以生成流程模板：
 
```text
你已经 5 次执行类似流程：
打开岗位页 → 提取 JD → 匹配简历 → 生成建议
是否保存为“岗位分析工作流”？
```
 
### 4. 数据闭环
 
未来训练目标识别、动作选择和错误恢复模型，需要真实操作轨迹。
 
---
 
## 隐私原则
 
任务日志可能包含敏感信息，因此默认：
 
- 本地保存
- 用户可删除
- 敏感字段脱敏
- 不默认上传截图
- 云端调用前显示将发送的上下文摘要
 
---
 
## 一句话总结
 
**AI 光标必须把“看见屏幕、协议化动作、可见执行、结果验证、事件留痕”作为底层能力：看见才能操作，协议才能安全，留痕才能复盘，重复留痕才能长出工作流。**
