## 用途

记录开发过程中**涌现但当前不实施**的想法与功能点。由于进度、优先级或成本原因暂不开发，但作为后续迭代的备选方向积累下来，避免遗忘。

- 与 `01_功能模块注册表` 互补：注册表登记「已纳入的模块」，本池子登记「涌现但暂不做的候选」。
- 与 `02_MVP开发规划与阶段路线` 相邻：开发规划决定「现在做什么」，本池子沉淀「以后可能做什么」，属于同一决策链。
- 正文数据全部维护在 xlsx 中，本 md 仅说明用途与字段含义，避免两处重复维护。

## 字段说明

| 字段 | 含义 |
|------|------|
| 序号 | 记录编号，可用「模块前缀-序号」形式（如 悬浮窗-01）便于按模块归组 |
| 提出日期 | 想法涌现的日期 |
| 模块类目 | 所属模块（可重复填写，同模块多个改进点即多行） |
| 想法/功能名称 | 简短命名 |
| 详细描述 | 具体想做什么 |
| 类型 | 关键闭环功能 / 非关键优化 / 体验打磨 / 探索 |
| 关联Cycle | 预期归属的产品 Cycle |
| 优先级 | P0–P3 |
| 预估成本 | 高 / 中 / 低 |
| 预期收益 | 高 / 中 / 低 |
| 状态 | 待评估 / 已采纳 / 开发中 / 已完成 / 暂缓 / 已放弃 |
| 决策原因 | 尤其记录「为什么现在不做」 |
| 备注 | 补充信息 |

## 使用约定

- **一个改进点一行**：模块类目只是可重复标签，不要用合并单元格归组；同一模块的多个功能点即多行，靠筛选/排序聚合。
- **状态是验收门（二值语义）**：与填色式开发进度一致 —— 「暂缓/待评估」等同未点亮，「已完成」等同已验收。进度或迭代空间用「决策原因/备注」文字说明，不要用颜色深浅表示完成百分比。
