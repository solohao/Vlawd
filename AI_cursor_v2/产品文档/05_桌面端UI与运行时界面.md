## 实现原则（UI.RUNTIME）

AI Cursor V2 的界面分成两类，不混用：

1. **App 主窗口页面**：Dashboard、Conversation Entry、Model Center、Workflows、Session Log。
2. **Runtime 运行时浮层**：Voice Capsule、Action Overlay、Compact Session Path、Session Graph。

App 主窗口用于配置和复盘；Runtime 浮层用于用户真实工作时监督 AI 操作电脑。

---

## App 主窗口页面（UI.APP）

首版实现采用 Electron/Vite 主窗口承载 framework-agnostic view model，不直接绑定 React/Vue：

- Dashboard：展示 AI 员工在线、对话入口、最近 Session、工作流、三角色状态。
- Conversation Entry：复用模块化对话入口策略，展示推荐、split、fallback、manual。
- Model Center：展示 Execution Brain、Record Notebook、Safety Engine 和模型下载位置。
- Workflows：先注册页面定义，后续接入从 Session 沉淀出的工作流。
- Session Log：先注册页面定义，后续承载完整 Session chunks、Graph、产物和导出。

当前 `npm run dev` 会启动真实 Electron 主窗口，并使用 preload IPC 连接 mock desktop runtime：

- Dashboard：显示 AI Employee、对话入口、三角色状态。
- Model Center：支持选择模型下载目录、mock 下载状态和健康检查。
- Runtime：显示 Voice Capsule、Action Overlay、Compact Session Path 和 Graph。
- 后续真实模型权重下载器、PCM/WebSocket 模型 server 与 BrowserView 实控会替换当前 mock bridge。

## Runtime 运行时浮层（UI.RUNTIME）

Runtime UI 采用渐进展开：

```text
常驻：顶部/角落 Voice Capsule
执行动作：目标高亮 + Action Card + Compact Session Path
查看详情：Session Drawer 切换完整 Timeline / Graph
```

面积规则：

- 常驻胶囊：只显示 AI 状态、对话入口、安全状态、暂停/取消/接管。
- 低风险动作：允许轻量倒计时执行，但必须可暂停或取消。
- 中风险动作：默认等待用户确认。
- 高风险动作：发送、分享、提交、删除、付款、权限变更必须强确认。
- Record Notebook 不作为默认大面板常驻，而是通过 compact path / drawer 暴露。

## Session Drawer 与 Graph View（UI.SESSION_GRAPH）

右下角抽屉不是单纯日志，而是 Session-first agent 架构的可视化前端。

首版合并策略：

- Now / Log 是同一个当前路径视图的简化态和详细态。
- Tree 可由当前步骤展开子任务，不单独做默认主视图。
- Graph 是独立视图，展示类似 git graph 的 fork / correction / merge。

Graph View 表达的是 AI 任务 Session，不是代码仓库 Git：

- 主线任务。
- 用户纠正分支。
- AI 尝试分支。
- 等待确认节点。
- 结果合并节点。

Runtime 默认只显示当前上下文附近的 compact path；完整 Graph 进入 Session 详情页。

## 品牌头像（UI.AVATAR）

AI Employee 头像是心理代理和状态体，不是装饰：

- 主窗口使用带底座形态，提供“AI 员工就绪”的情绪锚点。
- Runtime 使用 compact 形态，降低遮挡并保持状态识别。
- Light / Dark 共享同一造型，只做亮度、描边、阴影和背景适配。
