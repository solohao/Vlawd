## 核心判断

AI 光标的竞争力不只是"能操作电脑"，而是：

> **让用户放心地看着 AI 操作电脑，并在心理上获得指挥、代劳和解压感。**

v2 的核心体验升级：

> **全双工对话让"指挥 AI"像和员工说话一样自然——用户说话的同时 AI 就开始行动，打断和纠正不需要等待，每次纠正自动保留历史。**

---

## 全双工耳机协作

### V1 → V2 的体验跃升

V1 的"任务型准全双工"：

```text
用户说完 → AI 开始理解 → AI 执行 → AI 汇报
打断需要等 AI 检测到 → 延迟 1-3 秒
```

V2 的原生全双工：

```text
用户开始说话 → AI 同时理解并开始行动
用户打断 → AI < 200ms 内停止
用户纠正 → AI 立刻调整，无需重来
AI 持续汇报 → 用户可以随时插话
```

### 交互示例

```text
用户："帮我——"
AI（同时）："好的——"
用户："——搜一下远程前端岗位"
AI："正在搜。"
[AI 打开浏览器，输入搜索]
AI："找到了 8 个，前 3 个比较匹配。"
用户："先看第二个。"
AI（< 200ms）："好的，打开第二个。"
用户："嗯……这个要求太高了。"
AI："要我跳过这种，只看初级的吗？"
用户："对。"
AI："好的，重新筛选。"
```

全程无等待、无"请稍候"、无轮替间隙。

## 三层隔离感

沿用 v1 的三层隔离感设计：

### 1. 身体隔离

用户不需要亲自点、拖、填、切窗口，AI 光标承担身体层面的重复劳动。

### 2. 注意力隔离

AI 通过耳机实时汇报关键节点，用户不需要持续盯屏幕：

```text
"我打开了第 3 个结果。"
"这里需要登录，我暂停。"
"找到了 5 个候选。"
```

### 3. 心理隔离

用户不用亲自面对厌烦任务本身：

```text
不是我在痛苦地填网页
是我让 AI 员工去处理网页
```

v2 新增第四层：

### 4. 纠正无痛感

V1：纠正像"承认错误"——要重新解释
V2：纠正像"指导员工"——说一句就够

```text
"不对，右边那个" → AI 立刻调整，旧操作自动保留为 Session 分支
"回到刚才" → AI 从 Session 找到历史分支，恢复
"还是第一个方案好" → merge 回之前的分支
```

纠正不是失败，而是自然的工作方式。

## AI 什么时候应该说话

沿用 v1 的"好员工"原则，v2 全双工模型原生支持：

### 必须说

```text
开始执行任务
理解了用户纠正
任务进入新阶段
遇到不确定目标
页面卡住或失败
高风险动作前
需要用户输入
任务完成
```

### 可以少说

```text
普通点击
普通滚动
重复性动作
用户正在专注看屏幕时
```

### 全双工新增：自然附和

```text
用户说话间隙：
  AI："嗯"
  AI："好的"
  AI："我知道了"
```

附和让用户知道 AI 在听，不需要明确的"请继续"。

## 安全感的来源

### 1. 可见执行（沿用 v1）

所有动作在屏幕上可见发生。

### 2. 可打断（v2 增强）

```text
V1：打断延迟 1-3 秒
V2：打断延迟 < 200ms（全双工模型原生）
    安全控制词 < 50ms（本地规则引擎抢占）
```

### 3. 可纠正（v2 增强）

```text
V1：纠正后重走 Pipeline
V2：纠正在全双工模型内部实时调整
    同时 Session 自动 fork 新分支，旧历史保留
```

### 4. 可回溯（v2 新增）

```text
用户："刚才那个方案更好" → Session merge 回之前的分支
用户："帮我再做一次昨天那个" → Session 历史查询
用户查看右下角面板 → 看到完整操作历史（日志/树/拓扑）
```

### 5. 高风险确认（沿用 v1）

以下动作必须确认：提交、发送、付款、删除、覆盖、授权、发布、申请。

确认方式在 v2 中更自然：

```text
AI（全双工）："我准备点击'提交申请'，要继续吗？"
用户："确认。"
AI："好的，已提交。"
```

## Session 驱动的纠正机制

这是 v2 最重要的体验创新。

### 传统纠正

```text
用户："帮我搜明天北京天气"
AI 搜索 → 结果出来
用户："不对，是后天"
AI 重新搜索 → 之前的结果消失
```

### Session 纠正

```text
用户："帮我搜明天北京天气"
AI 搜索 → 结果记录为 Session 分支 A
用户："不对，是后天"
Session fork → 新建分支 B，分支 A 保留
AI 重新搜索 → 结果记录在分支 B
用户："算了还是明天的吧"
Session merge → 回到分支 A 的结果
```

右下角面板实时展示这个过程：

```text
树视图：
天气查询 ─┬─ 分支 A：明天 ← 当前
          └─ 分支 B：后天 [已放弃]
```

## 多 AI 员工的声音区分

沿用 v1 设计，v2 全双工模型增加了新能力：

```text
PersonaPlex 支持零样本语音克隆 + 角色定制
→ 可以给不同任务设定不同的 AI "人设"和声音

求职分析 AI：偏稳重
资料整理 AI：偏简短
写作 AI：偏温和
```

## 可监督执行协议

沿用 v1 设计，v2 中动作执行更自然：

```text
V1：每步展示 target/action/risk/expected_result
V2：全双工模型语音汇报关键信息
    低风险动作直接执行 + 简短语音确认
    高风险动作语音详细说明 + 等待确认
```

---

## 错误恢复原则

沿用 v1 的错误恢复原则，v2 新增 Session 回退：

```text
V1：观察状态 → 承认不确定 → 询问用户
V2：同上 + Session 自动保留错误分支
    用户可以说"回到上一步" → Session 回退到之前的 chunk
    错误分支保留 → 供未来分析和学习
```

---

## 一句话总结

**AI 光标 v2 的核心体验是"全双工对话 + Session 可回溯"：全双工让打断延迟从秒级降到 200ms，让纠正从"重新来过"变为"自然调整"；Session 让每次纠正自动 fork 新分支而非覆盖历史，让用户可以说"回到刚才那个版本"。安全感来自可见执行、即时打断、自然纠正、完整回溯和高风险确认五重保障。**
