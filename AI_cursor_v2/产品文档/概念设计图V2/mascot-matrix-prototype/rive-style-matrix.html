状态过渡矩阵（点格子：行→列，播放几何 morph 过渡）

Rive 状态机映射（运行时代码这样喂）：
• state ← duplex-runtime 状态：listening→就绪 · thinking→思考 · acting→执行 · paused→暂停 · interrupted→错误 · complete→完成
• mouthOpen(0..1) ← 说话音量（勾"口型"或点"模拟说话"看效果）
• lookX / lookY ← 鼠标/执行目标位置（勾"看向光标"，在预览区移动鼠标）
• 绿色对角线 = 该态待机循环（就绪/完成漂浮、思考=眼珠右→上→左扫视凝视、执行=前视微动、暂停=静止变暗、错误=抖动后稳定）。
• 其余格 = 从行态几何 morph 到列态：五官顶点重采样后逐点插值 + 缓动，形状连续变形（对比旧版 `index.html` 的交叉淡入）。
• 全程叠加呼吸/眨眼/看向光标等"混合输入"，这就是极简五官也能灵动的原因 —— 靠运动设计，不靠器官数量。
