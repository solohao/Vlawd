# Runtime 状态矩阵

Runtime 的状态显示必须同时包含文案、图标、颜色和可执行操作；颜色只作为辅助信号。

| 状态 | 用户文案 | 颜色 | 主要动作 | 禁止或降级动作 | 恢复路径 |
|---|---|---|---|---|---|
| Idle | 就绪 | 中性灰 | 开始对话、选择设备 | 暂停、接管 | 开始输入 |
| Connecting | 正在连接 | 信息蓝 | 取消连接 | 开始执行 | 成功进入 Listening；失败进入 Error |
| Listening | 正在聆听 | 品牌绿 | 静音、取消 | 重复开始录音 | 输入结束后进入 Thinking |
| Thinking | 正在思考 | 警示黄 | 暂停、取消、接管 | 批准不存在的动作 | 生成回答或 Proposal |
| Speaking | 正在回答 | 品牌绿 | 插话、暂停、取消 | 再次播放 | 插话进入 Interrupted |
| Interrupted | 已被打断 | 危险红 | 继续新约束、取消 | 自动重放旧输出 | 重新理解后进入 Thinking |
| Waiting confirmation | 等待确认 | 警示黄 | 确认、拒绝、接管 | 自动倒计时通过 | 精确批准当前 Proposal |
| Paused | 已暂停 | 中性灰 | 恢复、取消、接管 | 调度新动作 | 重新观察环境后 Resuming |
| Resuming | 正在安全恢复 | 信息蓝 | 取消、接管 | 复用旧坐标或旧权限 | Revalidate 后继续 |
| Error | 运行异常 | 危险红 | 重试、导出记录、取消 | 隐藏错误继续 | 从安全锚点重试 |
| Complete | 已完成 | 成功绿 | 查看证据、创建分支 | 继续写入原任务 | 新任务或分支 |
| Cancelled | 已取消 | 中性灰 | 查看记录、重新开始 | 恢复已终止动作 | 创建新 Session |

## 控制语义

- **暂停**：停止调度新动作，保留 Workspace、Session 与恢复锚点。
- **取消**：终止当前任务并释放资源，保留可审计 Session。
- **接管**：用户立即夺回系统或当前 Workspace 控制权，不代表 AI 执行动作。
- **确认**：只批准当前 Proposal 中描述的精确动作，不扩大权限。
- **拒绝**：阻止当前 Proposal 并记录理由。
- **恢复**：重新观察、重新校验权限与 postcondition 后从安全锚点继续。

## 风险级别

| 级别 | 例子 | UI 行为 |
|---|---|---|
| 低 | 读取公开页面、整理本地草稿 | 可自动执行，持续记录 |
| 中 | 打开外部链接、复制内容、修改草稿 | 提供撤销和明显状态 |
| 高 | 写文件、提交表单、发送消息 | 强确认，显示目标与影响 |
| 极高 | 付费、生产数据、永久删除、扩大权限 | 默认禁止或二次强确认 |
