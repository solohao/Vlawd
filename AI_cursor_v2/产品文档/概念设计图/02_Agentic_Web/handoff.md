# Agentic Web 概念边界与用户流程

Agentic Web 是长期 Living Project 平台蓝图。当前页面用于冻结对象语义和用户流程，不表示平台已经进入实现阶段。

## 核心对象

```text
Project
├─ Goal / Problem
├─ Plan / Solution
├─ Evidence
├─ Conflict / Counterexample
├─ Patch / Branch / Merge
├─ Run / Validation / Deployment
└─ Capsule / Permission / Lineage
```

Forum、Canvas、Diff、Run 与 Explore 必须是同一 Project Graph 的不同投影，不能形成互不一致的数据孤岛。

## 黄金路径

### 从发现到复用

```text
Explore 搜索问题
→ 打开 Project 权威视图
→ 查看 Evidence、风险与验证状态
→ 预览 Capsule 权限
→ 在受限环境 Run
→ 验证结果
→ Fork / Patch / Merge
```

### 从讨论到可执行项目

```text
Forum 提出问题
→ AI Expert 整理目标、约束与冲突
→ Draft-to-Graph
→ Canvas 检查节点和边
→ 建立 Project
→ Run / Validate
```

### 从待处理事项到审查

```text
Inbox 收到确认、Diff 或验证失败
→ 打开上下文
→ 查看影响和 Evidence
→ 批准、拒绝或请求修改
→ 事件回写 Project Graph
```

## 阶段边界

- Project、Graph、Evidence 语义可先稳定。
- Capsule 需要 Vlawd 第二个垂直案例验证后再产品化。
- Registry 必须晚于 Capsule，并以可验证能力和 Lineage 为基础。
- 平台 Feed、市场和大规模组织能力不得先于 Dogfood 证据。
