## 二、页面与后端对应关系核对（本次核心结论）

原则：页面渲染正常 ≠ 后端功能完成；标签（「本地运行中」「已就绪」等）不作为后端完成证据，除非来自真实服务状态。

### 2.1 各产品 Cycle 真实状态

| 层 | 对应页面 | 后端状态 | 说明 |
|---|---|---|---|

| Cycle 2 只读研究 | TaskWorkspace、任务卡 | **未开始** | `packages/main/src/executors/browser-mvp.ts` 仍是 `VirtualBrowserViewExecutor`（mock），无真实 WebContentsView/CDP、无 `OBS.STATE`/ref 生命周期、无 `ACTION.VERIFY`。 |

### 2.2 Cycle 1 前后端链路已确认「全链路打通，且前端有真实触发点」

逐段核对 `UI 控件 → hook → preload → main IPC → 运行时`，每个后端入口在前端都有触发点（不是只有后端 API 没法从界面点到）：

- 主窗口渲染 `packages/renderer/src/ui/DesktopApp.tsx`；首页麦克风区 `onStartTask` → 进入实时对话页 `packages/renderer/src/ui/pages/LiveConversationPage.tsx`。
- 实时对话页每个按钮都真的接到后端：连接 `convo.connect()`、麦克风开关 `convo.toggleMic()`（启动 VAD + renderer 内 Whisper 转写）、文字发送 `convo.submit()`、切 Provider `convo.setProvider()`、健康检查 `convo.checkHealth()`、暂停/取消/停止 `convo.preempt()`、继续 `convo.resume()`。
- 链路：`onClick` → `useConversation`(`packages/renderer/src/runtime/useConversation.ts`) → `desktopApi.conversation*` → `preload` 的 `ipcRenderer.invoke("conversation:*")` → `main.ts` 的 `ipcMain.handle("conversation:*")` → `DuplexConversationRuntime`。
- 反向：runtime 事件 → `webContents.send("conversation:event")` → `onConversationEvent` → 驱动界面状态 + TTS 朗报。
- preload 与 main.ts 两侧的 `conversation:` 8 个 channel（snapshot/connect/utterance/bargeIn/preempt/resume/setProvider/checkHealth）完全对齐，无孤儿入口。
- 启动时 `main.ts` 用 `createProvider(defaultPipelineProviderConfig)` 装好真 Provider（pipeline，接本地 Ollama `qwen2.5:7b-instruct`，见 `packages/main/src/model/dual-role-config.ts`），bayling-duplex 作为可切换候选。

**两个运行前提**：
1. 必须跑真正的 Electron app（`npm run dev`），不能在浏览器里开 `:4173`。所有对话按钮 gated 在 `available = !!window.aiCursorDesktop` 上，该对象只有 Electron preload 会注入；浏览器预览里不存在 → `available=false` → 按钮禁用/空操作（这也是浏览器预览截图看着像纯展示的原因）。
2. 「真模型回答」需本机装并运行 Ollama + `qwen2.5:7b-instruct`。没跑 Ollama 时闭环照样完整跑通（麦克风→Whisper→runtime→回复→TTS→可打断/抢占），但回复走离线 **Echo 兜底**（代码明确标注「非真实推理」），不是真模型输出（见 `packages/main/src/model/pipeline-duplex-provider.ts`）。

## 三、本次发现的问题：模型中心 Ollama 检测/安装入口在 V2 重做中被移除（功能回退）

用户反馈：旧模型中心有一套「检测用户电脑上有没有 Ollama → 没有就选定合适位置安装 → 有就直接调用」的界面（旧版 tab 为「模型配置 / 下载管理 / 运行日志」，含运行后端切换、安装/更新 Ollama、设备兼容性检测、三角色状态总览）。V2 重做后模型中心换成「配置 / 模型库」两个 tab，那套入口消失了。

核对结论（区分「能力没了」vs「界面入口没接」）：

### 3.1 仍然有效（自动跑，不用点）

- **检测有没有 Ollama + 有就直接调用**：`useModelCenter`(`packages/renderer/src/runtime/useModelCenter.ts`) 的 `useEffect` 在进入模型中心时自动跑 `modelProbeEnvironment` + `modelRefreshBackend` + `modelDetectOllamaInstaller`；探测到就直接用（截图顶部「Ollama 运行中」即出自这条链路）。
- **后端方法一个没删**：`packages/main/src/electron/main.ts` 里 `model:detectInstaller`、`model:locateInstaller`、`model:installOllama`、`model:refreshBackend`、`model:setBackend`、`model:setCustomEndpoint`、`model:useAsBrain`、`model:openInstallGuide`、`model:openStorageLocation` 全部还在注册，`useModelCenter` 也仍暴露它们。

### 3.2 界面入口没了（后端在，但用户点不到）

新版 `packages/renderer/src/ui/pages/ModelCenterPage.tsx` 只接了 4 个动作：`chooseStorageRoot`、`pull`、`cancelPull`、`removeModel`。以下旧功能在 UI 上无任何触发点：

- **「没有 Ollama → 选定位置静默安装」`installOllama` / `locateOllamaInstaller`**（正是用户关心的那步，现在界面上点不到）；
- 手动「重新检测」`refreshBackend`/`probe`（只在进页面时自动跑一次，无重跑按钮）；
- 「运行后端」切换 Ollama / LM Studio / 自定义端点 `setBackend`/`setCustomEndpoint`；
- 「用作执行大脑」`useAsBrain`、「打开存储目录」`openStorageLocation`、「安装引导」`openInstallGuide`。

已用 grep 确认：整个 `packages/renderer/src/ui/` 目录里没有任何地方调用上述方法。

### 3.3 一处会误导的文案

新页面写了「如未安装，将在需要时自动准备」，但渲染层没有任何地方调用 `installOllama`，后端也没在拉模型时自动触发安装——所以这句目前是**空承诺**。对一台还没装 Ollama 的电脑，V2 界面只能探测出「没装」，却给不了任何安装按钮。

### 3.4 结论

检测 + 直接调用还在（自动）；「没有就装到指定位置」的**后端能力在、但 V2 界面把入口砍了**，属于这次 UI 重做的功能回退。**尚未修复**（用户要求先做示意、暂不改 app 代码）。
