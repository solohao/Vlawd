## 背景

在 V2 之后的规划中，长期愿景扩展到 Agentic Web、Capsule、Living Project、Module Spec、高校知识论坛和 Post is Project。第一版整理曾将这些内容命名为 `AI_cursor_v3`。

这个命名不成立，因为其中大部分内容没有继承并替代 V2 的产品和实现，而是在更高层描述多个项目的关系。

---

## 决策

建立两条独立版本轴：

```text
Vlawd 产品：
AI_cursor_v1 → AI_cursor_v2 → 未来真正的 AI_cursor_v3

项目群蓝图：
Agentic_Web v1.0 → v1.1 → v2.0
```

`AI_cursor_v2` 继续作为 Vlawd 产品和当前代码的权威来源。Agentic Web 作为平行模块独立版本化，不再占用产品 V3 的语义。

---

## V2.10 合并内容

### 产品

- 明确 Vlawd 是 Agentic Web 的全双工个人终端和本地 Runtime；
- Vlawd 不负责提前实现整个网络、Registry 或开放平台；
- 产品生死假设仍是“监督 AI 做”比“自己动手做”更省。

### Runtime

- 明确 Conversation / Execution Brain、Record Notebook 和 Local Safety Preemption 的逻辑职责；
- Safety Preemption 固定本地实现；
- 职责拆分不代表 MVP 必须运行多个大模型。

### Task Workspace

- 多任务以任务卡和状态呈现，不以多个鼠标动画作为主体验；
- 补充 queued、active、waiting_confirmation、paused、failed、done、cancelled；
- 补充暂停、取消、接管、确认、拒绝、恢复和受控回滚；
- `target_view` 必须严格绑定 TaskCard、Workspace 和 Session。

### Session

- 从完整 Session 中提取可核验 Evidence；
- 生成 Execution Summary；
- 保存恢复锚点、未解决问题和环境重验证要求；
- 支持第二天打开并从明确分支继续；
- 明确 V2 Session Graph 不等于网络 Living Project。

### Dogfood

当前唯一 Golden Path：

```text
真实研究目标
→ 全双工讨论
→ 只读 BrowserView 可见执行
→ 用户途中纠正
→ 输出带来源结果
→ Session Evidence
→ 第二天继续
```

按三个周期交付：

1. 真实全双工入口；
2. 一个只读研究任务；
3. 可恢复 Session。

每周期只改善一个可感知结果，并保留 30–90 秒真实演示。

### 调优止损

- 当前 UI 目标为 U2，不追求 U3；
- 同一摩擦出现三次才进入视觉优化；
- 纯视觉工作不超过周期 10%；
- 单个纯视觉问题 30 分钟止损；
- 一个周期固定一个模型和 Provider；
- 候选模型先回放历史 Session。

## 路线调整

原路线将 Marketplace 和插件生态直接放入 Vlawd Phase 2 / Phase 3。V2.10 将其改为 deferred：

```text
Vlawd Golden Path
→ 第二垂直证据
→ Capsule draft
→ Registry
→ Post is Project
→ 再评估市场和插件生态
```

这一跨项目顺序由 `Agentic_Web/02_跨项目开发路径与阶段门.md` 管理。

---

## 代码影响

本次是产品和文档架构调整，不表示以下功能已经实现：

- 真实全双工 Provider；
- 严格 `target_view` 隔离；
- 完整任务控制状态机；
- 只读研究 Golden Path；
- Session Evidence 和恢复；
- 自动历史回放。

当前代码仍位于：

```text
AI_cursor_v2/app
```

后续继续在现有工程中做最小、连续实现，不复制为 V3 工程。

---

## 结论

> **V2.10 的目标不是扩张功能数量，而是把 Vlawd 的产品闭环重新收紧：先完成可监督、可恢复、可重复使用的个人研究任务，再让真实证据决定下一步。**
