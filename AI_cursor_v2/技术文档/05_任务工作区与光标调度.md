## 核心判断

桌面操作的根本约束是：

> 真实光标永远只有一个，硬件中断级别的排他资源。

CursorLease 调度器是 AI 光标解决"多 AI 任务共享一个光标"的核心方案。

v2 中 CursorLease 保持设计不变，新增**每个 Lease 绑定一个 Session**。

---

## CursorLease 调度器

沿用 v1 设计：

```text
CursorLease = {
  holder: "ai_task_001",
  acquired_at: timestamp,
  expires_at: timestamp,
  priority: 1-10,
  context: ExecutionContext
}
```

### 调度规则

```text
同一时间只有一个 Lease 持有者可以操作光标
用户随时可以抢回（优先级最高）
高优先级任务可以抢占低优先级
Lease 有 TTL，超时自动释放
任务暂停时释放 Lease
```

### v1.1 变更：Lease 绑定 Session

每个 CursorLease 关联一个 Session：

```text
CursorLease = {
  holder: "ai_task_001",
  session_id: "session_abc",     # v1.1 新增
  acquired_at: timestamp,
  expires_at: timestamp,
  priority: 1-10,
  context: ExecutionContext
}
```

当 Lease 切换时，对应 Session 也切换：

```text
Task A 持有 Lease → Session A 活跃
用户抢回 Lease → Session A 暂停，记录 chunk: "user_takeover"
Task B 获得 Lease → Session B 活跃
```

这保证了 Session 记录与实际执行的一致性。

## Task Card（任务卡）

沿用 v1 设计，v1.1 新增 Session 绑定：

```text
TaskCard = {
  id: "task_001",
  name: "搜索前端岗位",
  status: active | queued | paused | done,
  workspace: browser_context_1,
  session_id: "session_abc",    # v1.1 新增
  steps: [...]
}
```

### 任务卡的生命周期

```text
用户说"帮我搜岗位" → 创建 TaskCard + 创建 Session
TaskCard 排队 → Session 状态: pending
TaskCard 获得 Lease → Session 状态: active
TaskCard 暂停 → Session 状态: paused
TaskCard 完成 → Session 状态: completed
TaskCard 取消 → Session 状态: cancelled
```

### 多任务卡的 Session 管理

```text
TaskCard A (active)  → Session A (active)
TaskCard B (queued)  → Session B (pending)
TaskCard C (paused)  → Session C (paused)
```

Session 之间相互独立，互不干扰。

---

## Workspace（工作区）

沿用 v1 的工作区概念：

- 浏览器上下文组（一组标签）
- 文件和资料区
- 任务面板

v1.1 新增：

- 每个工作区绑定一个或多个 Session
- 工作区切换触发 Session 上下文切换
- 任务面板自动展示当前工作区关联的 Session 视图

---

## 多光标安全约束

沿用 v1 设计：

```text
Rule 1：同一时刻只有一个执行器控制真实光标
Rule 2：用户操作立刻抢占所有 AI Lease
Rule 3：Lease 超时自动释放
Rule 4：高风险任务不能被低优先级任务打断
Rule 5：所有 Lease 操作写入当前 Session
```

v1.1 新增：

```text
Rule 6：Lease 切换时，记录引擎写入 session.fork() 或 session.pause()
Rule 7：用户抢回 Lease 时，Session 记录 "user_takeover" chunk
```

---

## 一句话总结

**CursorLease 调度器保持 v1 设计不变——真实光标只有一个，通过 Lease 排他调度。v1.1 的核心更新是每个 Lease 绑定一个 Session，保证"谁在操作光标"和"Session 记录什么"始终一致。任务卡的生命周期与 Session 状态同步，工作区切换自动切换 Session 上下文。**
