# AI Cursor V2 代码执行清单

目标：按 Phase 0-4 落地一个可验证的全双工语音监督 + 可见执行 + Session 留痕工程骨架。每阶段必须有可运行验证。

## Phase 0：工程骨架

- [x] 建立 app/ npm workspace 工程。
- [x] 建立 packages/shared、packages/main、packages/renderer。
- [x] 定义 Action、Session、DuplexModel 的共享类型。
- [x] 建立主进程/渲染层/共享包的 TypeScript 构建。
- 验证：npm run typecheck 通过；包之间类型引用正常。

## Phase 1：Mock 模型到可见执行闭环

- [x] 实现 MockDuplexModelProvider，模拟全双工模型输出 speech/action/state。
- [x] 实现 AgentLoop：model proposal -> safety policy -> executor -> session chunk。
- [x] 实现 MockSystemExecutor，只模拟动作，不真实控制电脑。
- [x] 实现 JSONL Session append-only 记录。
- [x] 实现安全等级：safe / confirmation_required / blocked。
- 验证：单元测试覆盖动作执行、安全确认、Session 记录；demo 能跑完一次搜索/表单模拟流。

## Phase 2：语音监督占位与安全抢占通道

- [x] 实现 停/暂停/别动/取消/退回/继续 关键词抢占通道。
- [x] 抢占逻辑独立于模型 provider，不依赖 VAD turn-taking。
- [x] 暂停/取消能阻止 executor 继续执行后续动作。
- [x] 提供 AudioSupervisionRuntime 作为语音监督状态占位。
- 验证：测试证明暂停/取消优先于模型动作；暂停后不继续执行剩余动作。

## Phase 3：DuplexModelProvider 可插拔模型层

- [x] 定义并使用统一 DuplexModelProvider 接口。
- [x] 实现 ProviderRegistry，支持 mock / stub / glm4-voice-stub / bayling-stub / personaplex-stub。
- [x] AgentLoop 不绑定具体模型实现。
- [x] 实现 raw model JSON -> ActionProposal 的解析/校验边界。
- 验证：同一 AgentLoop 能用 MockProvider 和 StubProvider 替换运行；不合法动作会在进入 executor 前被拒绝。

## Phase 4：浏览器场景 MVP 骨架

- [x] 实现 VirtualBrowserViewExecutor 虚拟浏览器执行器。
- [x] 支持搜索、筛选、表单填写、overlay label 等 MVP 动作。
- [x] Session 中记录每步动作、预期结果和安全等级。
- 验证：场景测试覆盖搜索、筛选、表单；Session 中能看到每步动作、预期结果和安全等级。

## 当前本地运行命令

- `npm run demo`：先构建 workspace `dist`，再运行 Mock Agent demo。
- `npm run dev`：启动 Vite dev server 与 Electron 主窗口，渲染 Dashboard / Model Center / Runtime mock UI。
- `npm run dev:smoke`：构建后短暂启动 Electron，用于验证 preload、IPC 与渲染入口可用。
- 当前 Electron UI 已接入模型下载目录选择、mock 下载状态、健康检查、对话入口连接、暂停/取消和 Runtime 动作模拟；真实模型权重下载器与真实 PCM/WebSocket 模型 server 仍需在后续替换 mock bridge。
