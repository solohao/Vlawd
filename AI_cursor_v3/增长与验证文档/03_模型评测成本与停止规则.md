## 核心原则

> **不寻找全世界最好的模型，只选择当前场景的合格模型。**

模型选择服务于 Golden Path，不应成为独立长期研究项目。

---

## 合格 Baseline（EVAL.BASELINE）

一个模型达到以下条件即可成为当前 Baseline：

- 能运行当前场景；
- 延迟进入体验可接受范围；
- 中文或目标语言理解合格；
- 输出能够进入结构化 Action / Project 边界；
- 成本低于任务预算；
- 许可证和部署方式可接受；
- 失败能够被安全层阻止；
- 连续运行稳定。

选择后至少冻结一个完整开发周期，推荐稳定使用 4–6 周。除非完全无法运行或出现安全、许可证和成本阻塞，不在周期内更换。

---

## 模型职责拆分（EVAL.ROUTE）

```text
实时语音模型：
  听、说、turn-taking、插话

Execution Brain：
  目标理解、规划、ActionProposal

Record Notebook：
  Session 摘要、Project Draft、长期记忆

Safety：
  本地规则和状态机，不使用模型替代
```

一个模型可以早期承担多个角色，但接口必须保持职责可拆分。

用户界面不需要持续展示具体模型名称，可以提供：

- 均衡；
- 低成本；
- 隐私优先；
- 高质量。

高级用户再查看底层配置。

## Session 回放评测（EVAL.REPLAY）

新模型先在历史 Session 上离线评测，不直接替换当前体验。

### 数据集

- 成功 Golden Sessions；
- 插话和纠正案例；
- 不确定目标；
- 安全阻止案例；
- 失败恢复案例；
- 不同网页和环境；
- 长 Session 压缩案例。

### 指标

- 任务成功率；
- 结构化输出有效率；
- 错误动作数；
- 安全违规；
- 首音延迟；
- 插话停止延迟；
- 总完成时间；
- 用户纠正次数；
- 推理成本；
- 失败可解释性。

### 评测流程

```text
候选模型
→ 离线重放
→ 与 Baseline 比较
→ 只在明显优势维度进入小流量
→ 真实 Dogfood
→ 决定保留或回退
```

---

## 模型替换规则

只有满足以下至少一项且没有造成关键退化才替换：

- 任务成功率有明显提高；
- 成本显著下降；
- 延迟显著下降；
- 安全或结构化输出显著改善；
- 解决 Baseline 无法完成的核心场景；
- 许可证或部署风险显著降低。

不替换的理由：

- 新模型更热门；
- 单个 Demo 看起来更聪明；
- 参数量更大；
- 某项非核心 Benchmark 更高；
- 界面可以增加一个新选项；
- 尚未在历史 Session 上验证。

模型评测集中到固定窗口，建议每月最多一次。

---

## 成本预算（EVAL.BUDGET）

需要同时控制四类成本：

### 推理成本

- 每分钟语音；
- 每次研究任务；
- 每次 Project Draft；
- 每次回放评测；
- 每个活跃用户每月。

### 计算资源

- GPU 显存；
- 并发；
- 队列等待；
- Worker 利用率；
- 空闲资源。

### 开发注意力

- 模型接入时间；
- 调参时间；
- 评测维护；
- 多模型 UI；
- 部署和故障处理。

### 用户监督成本

- 解释次数；
- 插话次数；
- 确认次数；
- 失败重做；
- 等待时间。

用户监督成本是产品最重要的成本，不能只优化 Token。

## 云端共享推理

校园或 Web MVP 第一版：

```text
一个固定模型
+ 一个 GPU Worker
+ 一个 Voice Gateway
+ Session 隔离
+ 并发上限
+ 排队
+ 用户额度
+ 监控
```

只有满足真实并发门槛后才建设：

- 多 Worker 调度；
- 弹性扩容；
- 模型路由；
- 容错和迁移；
- 更复杂计费。

## 停止规则（EVAL.STOP）

### 模型调优停止

当前 Baseline 达到验收线且连续三次完成 Golden Path后，停止继续优化模型，回到产品闭环。

### 功能停止

功能不能改善当前指标、没有真实 Session 证据，或只为未来平台服务时，停止开发并进入 Idea Vault。

### UI 停止

状态清楚、任务可完成且不易误操作后停止。美观问题使用 Dogfood 文档的 10% 预算和 30 分钟规则。

### 路径停止

一个验证周期结束后，若核心假设没有改善趋势：

- 缩小场景；
- 改用更低成本模态；
- 暂停该路径；
- 不用新增功能掩盖需求问题。

### 规划停止

当下一周期的用户体验、验收标准和 Not Now 已明确后，停止继续规划。长期问题不要求在当前阶段得到答案。

## 一句话总结

**模型是可替换基础设施：先冻结合格 Baseline，用真实 Session 回放比较，只在成功率、成本、延迟或安全出现明确收益时替换，并给调优、算力和注意力设置预算。**
