# 00 · 技术文档总览

## 一句话定义

**AI 光标的技术系统是一套以耳机语音/快捷键为入口、以 Voice Interaction Runtime 为协作中枢、以本地快脑 + 云端强脑 + 工作流记忆为推理分层、以屏幕感知为上下文、以结构化动作协议为核心、以鼠标键盘/浏览器/窗口执行器为手、以任务日志和安全策略为边界的桌面 Agent 执行架构。**

工程实现上应优先采用 **模块化单体 + DDD 边界 + Hexagonal Architecture + 稳定协议 + 插件化扩展**，让代码对人类和 AI 都易读、易改、可测试，并避免版本迭代时频繁重构。

---

## 技术目标

### 1. 可见执行

AI 不只输出文本，而要驱动真实桌面动作：

- 移动光标
- 点击
- 右键
- 双击
- 拖拽
- 滚动
- 输入
- 快捷键
- 窗口/标签操作

### 2. 实时交互

高频指令必须低延迟：

- 停止
- 暂停
- 继续
- 点编号
- 滚动
- 返回

耳机反馈必须可中断：

- AI 播报时用户可打断
- 停止指令本地抢占
- 高风险动作语音确认
- 关键进度短句汇报

### 3. 可监督安全

系统必须支持：

- 动作前目标高亮
- 高风险确认
- 可暂停/继续/取消
- 执行日志
- 回退和错误恢复

### 4. 低成本运行

高频、低风险、可验证动作本地化，复杂推理才调用云端强模型。

```text
本地快脑：ASR / TTS / VAD / 快捷命令 / 元素定位 / 动作校验 / 工作流执行
云端强脑：复杂规划 / 长上下文理解 / 写作 / 困难失败恢复
工作流记忆：成功路径沉淀后，下次尽量本地复用
```

### 5. 工作流沉淀

每一次执行都要留痕，为未来从“临时指挥”升级为“可复用工作流”打基础。

## 核心模块

### 1. Interaction 模块

负责用户输入输出：

- 全局快捷键
- 麦克风录音
- 本地唤醒
- 流式 ASR
- TTS 语音反馈
- 耳机输入输出设备选择

### 1.5 Voice Interaction Runtime 模块

负责耳机协作：

- VAD
- ASR
- TTS
- 本地抢占词
- 可中断播报
- 任务型准全双工状态机
- Interaction Policy
- 语音与任务日志绑定

### 2. Command Router 模块

判断用户命令属于哪一类：

- 快速控制命令
- 局部光标指令
- 中层网页/窗口指令
- 长任务规划指令
- 安全确认响应

### 3. Observation 模块

负责获取当前电脑状态：

- 屏幕截图
- OCR 文本
- 鼠标位置
- 当前窗口
- 浏览器标签
- DOM / Accessibility 树
- 可点击元素候选

### 4. Planning 模块

负责把用户意图转成计划：

- 任务拆解
- 下一步动作选择
- 不确定性判断
- 安全风险判断
- 失败恢复策略

### 5. Action Protocol 模块

定义 AI 输出的结构化动作。

AI 不直接输出自然语言，也不直接输出裸坐标，而输出：

```text
目标 + 动作 + 参数 + 风险级别 + 预期结果
```

### 6. Executor 模块

把动作协议翻译为真实系统操作：

- 鼠标事件
- 键盘事件
- 浏览器 API
- 窗口管理
- 剪贴板
- 文件读写

### 7. Safety & Policy 模块

负责：

- 高风险动作拦截
- 用户确认
- 权限边界
- 敏感字段保护
- 操作撤销
- 错误停止

### 8. Trace & Workflow 模块

负责记录：

- 用户原始指令
- 屏幕上下文
- AI 计划
- 执行动作
- 执行结果
- 用户纠错
- 最终产出

### 9. Workspace & Scheduler 模块

负责多 AI 工作区和单真实光标调度：

- 任务工作区
- 虚拟光标状态
- 真实光标租约
- 前台执行排队
- 用户中断抢占
- 拖拽资料输入
- 工作区上下文隔离

### 10. Voice Model Selection 模块

负责按阶段选择语音技术：

- 本地 ASR
- 本地 TTS
- VAD
- LiveKit/WebRTC 远程入口
- 端到端全双工模型
- 多角色音色

### 11. Personalization Layer 模块

负责用户偏好和可监督自配置：

- 习惯用语
- 工作流快捷语
- 网站规则
- 工作区预设
- 安全策略覆盖
- 交互风格
- 配置变更确认
- 配置变更留痕

### 12. Architecture Governance 模块

负责长期可维护性：

- bounded context 划分
- 领域对象与协议版本
- ports/adapters 边界
- 插件合同测试
- ADR 决策记录
- AI-readable 模块 README
- 跨模块依赖约束

### 13. Local Model Runtime 模块

负责低延迟本地推理：

- 短命令识别
- 本地动作选择
- 元素候选排序
- 动作结果验证
- 工作流模板执行
- 云端调用降级与异步化

## 技术实现优先级

### P0：可跑通原型

- 全局快捷键
- 麦克风输入
- ASR
- 本地 TTS 短反馈
- 本地停止/继续/点编号命令
- 屏幕截图
- 元素编号
- 鼠标移动/点击/滚动/输入
- 停止/继续/纠错
- 动作日志
- 模块化单体目录结构
- 核心协议版本化

### P1：浏览器增强

- Chrome 扩展
- DOM 读取
- 标签管理
- 表单识别
- 网页结构化摘要
- 标签组操作
- 任务型准全双工耳机交互
- 可中断 TTS 播报
- executor / observer 插件边界

### P2：工作流沉淀

- 轨迹回放
- 相似任务识别
- 流程模板生成
- 用户可编辑工作流
- 多工作区任务上下文
- 习惯用语与工作流快捷语
- 可监督偏好配置
- event-sourced trace
- golden trace tests

### P3：模型优化

- 本地命令识别小模型
- 元素定位模型
- 动作选择模型
- 失败恢复模型
- model provider 插件化
- contract tests
- 本地 verifier
- 云端 thinking 后台化
- 成功路径本地工作流化

---

## 一句话总结

**AI 光标的技术关键不在“训练一个会点鼠标的模型”，而在“本地快脑处理实时交互 → 云端强脑处理复杂规划 → 工作流记忆复用成功路径 → 观察屏幕 → 结构化动作 → 可见执行 → 结果反馈 → 安全留痕”的闭环。**
