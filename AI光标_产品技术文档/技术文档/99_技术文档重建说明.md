# 99 · 技术文档重建说明

## 为什么需要重建

随着文档连续补充，技术文档里出现了两个容易混淆的“架构设计”：

```text
01_架构设计.md
08_面向AI可维护的架构设计.md
```

它们并非完全重复，但命名容易让读者误解。

最新重建后，二者职责分离：

```text
01_系统运行时架构.md
  讲系统运行时怎么流动：
  用户输入 → 语音交互 → 观察 → 规划 → 动作协议 → 执行 → 反馈 → 留痕

08_工程架构与AI可维护性.md
  讲代码仓库怎么组织：
  模块化单体 → DDD → Hexagonal → 稳定协议 → 插件化 → AI-readable conventions
```

## 最新架构主线

重建后的主线是：

```text
产品层：
耳机协作式 AI 光标
→ 可监督桌面员工
→ 多工作区
→ 可监督自配置

运行时层：
Voice Interaction Runtime
→ Observation
→ Planning
→ Action Protocol
→ Safety
→ Execution Scheduler
→ Trace

模型层：
本地快脑
→ 云端强脑
→ 工作流记忆

工程层：
模块化单体
→ DDD bounded contexts
→ Hexagonal ports/adapters
→ 稳定协议
→ 插件化
→ AI-readable conventions
```
