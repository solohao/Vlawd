# 02 · 动作协议与执行器

## 核心原则

AI 不应直接输出：

```text
“我去点一下右边的按钮”
```

也不应直接输出裸坐标：

```json
{"x": 842, "y": 317}
```

而应输出结构化动作：

```json
{
  "action": "mouse.click",
  "target": {
    "type": "screen_element",
    "ref": "element_3",
    "description": "第一个搜索结果"
  },
  "button": "left",
  "safety": "safe",
  "expected_result": "打开搜索结果详情页"
}
```

执行器再把目标映射到真实坐标或系统 API。

---

## 动作协议分层

### 1. Mouse Actions

```json
{
  "action": "mouse.move_to",
  "target": {"ref": "element_1"},
  "duration_ms": 300
}
```

```json
{
  "action": "mouse.click",
  "target": {"ref": "element_2"},
  "button": "left",
  "click_count": 1
}
```

```json
{
  "action": "mouse.drag",
  "source": {"ref": "tab_3"},
  "destination": {"ref": "window_left"},
  "duration_ms": 800
}
```

```json
{
  "action": "mouse.scroll",
  "target": {"ref": "current_page"},
  "direction": "down",
  "amount": "medium"
}
```

### 2. Keyboard Actions

```json
{
  "action": "keyboard.type",
  "text": "AI 光标产品文档"
}
```

```json
{
  "action": "keyboard.hotkey",
  "keys": ["ctrl", "l"]
}
```

```json
{
  "action": "keyboard.press",
  "key": "enter"
}
```

### 3. Browser Actions

浏览器动作可以由浏览器 API 执行，也可以由鼠标键盘模拟执行。

```json
{
  "action": "browser.new_tab"
}
```

```json
{
  "action": "browser.open_url",
  "url": "https://example.com"
}
```

```json
{
  "action": "browser.group_tabs",
  "tabs": ["tab_1", "tab_2", "tab_3"],
  "group_name": "求职资料"
}
```

### 4. Window Actions

```json
{
  "action": "window.arrange",
  "layout": "two_columns",
  "left": "browser_window_1",
  "right": "document_window_1"
}
```

```json
{
  "action": "window.focus",
  "target": "browser_window_1"
}
```

### 5. Clipboard Actions

```json
{
  "action": "clipboard.copy_selection"
}
```

```json
{
  "action": "clipboard.paste"
}
```

### 6. Control Actions

```json
{
  "action": "task.pause",
  "reason": "等待用户确认"
}
```

```json
{
  "action": "user.ask",
  "question": "这里需要提交表单，要继续吗？",
  "options": ["继续", "取消", "我来接管"]
}
```

## 动作对象结构

每个动作建议包含：

```json
{
  "id": "step_001",
  "task_id": "task_abc",
  "action": "mouse.click",
  "target": {},
  "params": {},
  "safety": "safe | confirmation_required | blocked",
  "expected_result": "",
  "fallback": "",
  "timeout_ms": 5000
}
```

---

## 安全等级

### safe

可以直接执行：

- 滚动
- 切标签
- 打开网页
- 点击普通链接
- 输入草稿
- 复制内容

### confirmation_required

必须确认：

- 提交
- 发送
- 删除
- 覆盖
- 授权
- 付款
- 发布
- 申请

### blocked

默认禁止：

- 绕过验证码
- 自动输入敏感密码
- 未确认付款
- 未确认删除大量文件
- 未确认发送外部消息

---

## 执行器职责

执行器不是简单调用系统 API，它还要负责：

1. 目标定位
2. 动作前高亮
3. 执行动画/光标移动
4. 执行后观察
5. 判断是否达到预期
6. 失败重试或询问用户
7. 写入日志

---

## 执行反馈

每个动作完成后返回：

```json
{
  "step_id": "step_001",
  "status": "success | failed | uncertain | interrupted",
  "observation": "页面已打开岗位详情",
  "screenshot_ref": "screen_001",
  "error": null,
  "next_required": "continue | observe | ask_user"
}
```

---

## 低层光标动作的价值

低层动作看起来“笨”，但它是产品信任机制：

- 用户看得见
- 用户能理解
- 用户能纠错
- 用户能接管
- 跨软件通用

因此，不应把低层动作视为失败，而应把它作为稳定兜底层。

---

## 一句话总结

**AI 光标的动作协议应让模型输出“我要对哪个目标做什么动作”，而不是直接输出坐标；执行器负责把语义动作变成可见、可控、可回滚的真实电脑操作。**
