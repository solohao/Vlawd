# 03 · 屏幕感知与任务留痕

## 核心判断

AI 光标要能替用户操作电脑，必须先知道当前屏幕上有什么。

但屏幕感知不能只依赖截图，也不能只依赖 DOM。更稳妥的方式是多源融合：

```text
Screenshot
+ OCR
+ DOM
+ Accessibility Tree
+ Window State
+ Cursor State
+ Clipboard
```

---

## 屏幕感知输入

### 1. 截图

截图适合：

- 跨软件统一观察
- 视觉布局判断
- 鼠标目标高亮
- 用户监督和回放

缺点：

- 结构化程度低
- 成本高
- 需要视觉模型或 OCR

### 2. OCR

OCR 适合：

- 识别按钮文字
- 识别网页标题
- 识别表单标签
- 支持屏幕标号

### 3. DOM

DOM 适合网页：

- 获取链接
- 获取按钮
- 获取输入框
- 获取表单结构
- 获取页面文本

缺点：

- 只适用于网页
- 有些网页结构混乱
- Shadow DOM / Canvas / 跨域 iframe 可能有问题

### 4. Accessibility Tree

适合桌面和浏览器可访问性元素：

- 按钮
- 输入框
- 菜单
- 标签页
- 窗口控件

它比 OCR 更结构化，比 DOM 更跨应用。

### 5. Window State

需要知道：

- 当前前台窗口
- 所有打开窗口
- 窗口位置和尺寸
- 浏览器标签页
- 分屏布局

---

## 目标候选生成

Observation 模块不应只把原始截图丢给模型，而应生成目标候选：

```json
[
  {
    "id": "element_1",
    "type": "input",
    "label": "搜索",
    "bounds": [120, 80, 620, 120],
    "source": "accessibility",
    "confidence": 0.92
  },
  {
    "id": "element_2",
    "type": "button",
    "label": "筛选",
    "bounds": [760, 210, 830, 248],
    "source": "ocr",
    "confidence": 0.81
  }
]
```

模型再根据用户指令选择目标。

---

## 屏幕标号

屏幕标号是 MVP 的关键交互。

流程：

```text
观察屏幕
↓
生成可点击/可操作元素
↓
给元素编号
↓
在屏幕上 overlay 编号
↓
用户说“点 3”
↓
执行器点击 element_3
```

标号要支持：

- 点击
- 右键
- 拖拽
- 输入
- 滚动区域
- 标签页
- 窗口

## 任务留痕

每个任务都应记录为事件流，而不是只保存最终结果。

### TaskRun

```json
{
  "task_id": "task_001",
  "goal": "帮我找 10 个适合我的岗位",
  "status": "running",
  "created_at": "2026-06-16T00:00:00Z",
  "input_mode": "voice",
  "workspace": "browser_window_1"
}
```

### StepEvent

```json
{
  "event_id": "event_001",
  "task_id": "task_001",
  "type": "action_executed",
  "user_command": "打开第一个结果",
  "observation_ref": "obs_001",
  "action": {
    "action": "mouse.click",
    "target": "element_2"
  },
  "result": "success",
  "timestamp": "2026-06-16T00:00:01Z"
}
```

### UserCorrection

```json
{
  "type": "user_correction",
  "text": "不是这个，点右边那个",
  "previous_step": "event_001",
  "resolution": "target_changed"
}
```

---

## 为什么留痕重要

### 1. 安全复盘

用户可以知道 AI 做过什么。

### 2. 错误恢复

出错后可以回到上一步上下文。

### 3. 工作流沉淀

多次类似任务后，可以生成流程模板。

### 4. 数据闭环

未来训练目标识别、动作选择和错误恢复模型，需要真实操作轨迹。

---

## 从留痕到工作流

当同类任务重复出现时，系统可以提示：

```text
你已经 5 次执行类似流程：
打开岗位页 → 提取 JD → 匹配简历 → 生成建议
是否保存为“岗位分析工作流”？
```

工作流不是用户预先配置，而是从实际指挥中生长出来。

---

## 隐私原则

任务日志可能包含敏感信息，因此默认：

- 本地保存
- 用户可删除
- 敏感字段脱敏
- 不默认上传截图
- 云端调用前显示将发送的上下文摘要

---

## 一句话总结

**AI 光标必须把“看见屏幕”和“记录执行”作为底层能力：看见才能操作，留痕才能安全，重复留痕才能长出工作流。**
