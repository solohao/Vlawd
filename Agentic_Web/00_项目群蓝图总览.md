## 长期愿景

> **构建一个可执行、可验证、可演化的 Agentic Web。**

更完整的定义：

> 让人通过自然交互调度本地与云端 Agent 完成现实任务，并把每次执行沉淀为可验证、可复用、可持续进化的能力与项目。

```text
今天的互联网：
人阅读页面、点击界面、手工完成操作

目标中的 Agentic Web：
人表达目标
→ Agent 发现知识、能力与服务
→ 优先选择 API、MCP、CDP 或语义界面等最可靠执行入口
→ 在明确权限内执行
→ 人可以监督、打断和纠正
→ 结果形成 Evidence
→ 成功经验沉淀为 Capsule / Living Project
→ 可执行 Project 在干净环境中被真实运行和验证
→ 其他人和 Agent 可以复用、验证、Fork 与改进
```

## 本模块是什么

`Agentic_Web/` 是整个项目群的长期蓝图和跨项目控制面，负责回答：

- 项目群最终要形成什么系统；
- Vlawd、Capsule、Registry、Post is Project、Module Spec 和高校论坛分别处于哪一层；
- 各项目之间依赖什么共同对象和接口；
- 当前应该先验证哪条路径；
- 什么时候允许提取协议、建设 Registry 或平台；
- Project 何时可以进入隔离验证、临时 Preview 或稳定部署；
- Runtime 何时可以通过语义桌面交互补足结构化接口，以及如何证明动作真的成功；
- 各项目版本怎样独立演进并保持兼容。

它不是某个具体产品的新版本，也不直接拥有 Vlawd 的详细产品设计。

## 两条独立版本轴

### 产品演进轴

```text
AI_cursor_v1
→ AI_cursor_v2
→ 未来真正的 AI_cursor_v3
```

这一轴只描述 Vlawd / AI 光标产品本身。新主版本必须继承并替代上一版本的产品定义、交互、架构、实现和迁移路径。

当前权威产品版本仍是：

```text
AI_cursor_v2/
AI_cursor_v2/app
```

只有 Vlawd 产品本体发生不可兼容的架构升级时，才创建真正的 `AI_cursor_v3/`。

### 项目群蓝图轴

```text
Agentic_Web v1.0
→ v1.1
→ v1.2
→ v2.0
```

这一轴描述项目群关系、共享对象、长期平台和跨项目路线。它可以在 Vlawd 仍处于 v2.x 时独立迭代。

### 版本关系

```text
项目版本 != 项目群版本

Agentic Web Blueprint v1.2
可以同时引用：
  Vlawd v2.12
  Capsule Protocol draft v0.1
  Module Spec concept v0.1
  Campus Lab concept v0.1
```

不存在要求所有项目同步升级的“全局统一版本号”。

## 权威来源

| 内容 | 权威目录 | 本模块只负责 |
|------|---------|-----------|
| Vlawd 产品定位、用户、交互、Runtime、Session、UI、MVP | `AI_cursor_v2/` | 记录其在项目群中的角色、依赖和阶段状态 |

| Capsule / Living Project 共享对象 | `Agentic_Web/共享架构与协议/` | 在跨垂直证据出现前维护概念草案 |
| 可执行项目、隔离环境和部署验证 | `Agentic_Web/共享架构与协议/03_可执行项目与部署验证.md` | 定义跨层职责、对象、权限和阶段门，不提前建设云平台 |

| Module Spec | 未来独立模块或仓库 | 记录启动条件和与共享对象的映射 |
| 高校知识论坛 | 未来独立模块或仓库 | 记录启动条件和平台依赖 |

原则：

> **产品细节只在产品文档中定义；项目群蓝图只引用，不复制。**

## 项目群一句话

**Vlawd 是 Agentic Web 的全双工个人终端；可验证环境交互让 Runtime 在缺少结构化接口时安全操作软件；Session 记录真实协作；Capsule 让能力可移植；Post is Project 让问题、方法和能力持续演化并在受控环境中接受真实验证。**

## 版本管理规则

### 稳定目录

目录表示模块身份，不携带版本后缀：

```text
Agentic_Web/
Capsule_Protocol/
Module_Spec/
```

版本记录在模块总览、各文档顶部和变更记录中。这样可以避免每次版本升级复制整套目录。

### 独立版本

```text
Vlawd 产品变化：
  只更新 AI_cursor_v2 或未来真正的 AI_cursor_v3

Agentic Web 项目关系变化：
  只更新 Agentic_Web

Capsule Schema 变化：
  只更新未来 Capsule Protocol 模块

跨项目兼容关系变化：
  更新 01_模块注册与版本矩阵
```

### 版本号

```text
主版本：
模块职责、对象模型或兼容边界发生破坏性变化

次版本：
模块内目标、接口、阶段或验证标准发生兼容性变化

修订：
不改变决策含义的文字、链接和示例修正
```

### 修改流程

```text
1. 确认内容属于哪个权威模块
2. 只修改该模块的权威文档
3. 更新文档变更记录和模块版本表
4. 若影响跨项目关系，再更新 Agentic Web 版本矩阵
5. 若影响开发顺序，再更新跨项目阶段门
6. 不在两个模块中复制同一份详细设计
```

## 当前执行焦点

项目群蓝图允许多条长期路径，但当前唯一产品主线仍由 Vlawd v2.x 管理：

> **通过 Vlawd 自我 Dogfood，证明“全双工讨论 / 研究 → 可见执行 → 可恢复 Session → 可继续项目”的个人闭环。**

Agentic Web 当前只负责记录后续启动条件，不要求同时建设：

- 通用 Agent 网络协议；
- Capsule Protocol 正式标准；
- Registry；
- Post is Project 平台；
- 通用多云部署平台和每项目永久 VM；
- 开放社区和传统 Feed；
- 共享 GPU 池；
- 自研耳机硬件；
- 商家支付与交易网络。

## 一句话总结

**`AI_cursor_v2` 是正在迭代的 Vlawd 产品，`Agentic_Web` 是独立迭代的项目群蓝图；前者负责把一个产品做深，后者负责说明多个项目为什么存在、怎样连接，以及何时才允许启动。**
