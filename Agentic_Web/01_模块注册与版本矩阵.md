## 状态定义

| 状态 | 含义 |
|------|------|
| active | 当前允许进入设计或实现 |
| validating | 正在通过 MVP 获取证据 |
| planned | 已定义，但未达到启动条件 |
| optional | 只有真实需求出现后才进入 |
| deferred | 明确延后 |
| deprecated | 已废弃，不再采用 |

## 独立版本矩阵

| 模块 | 当前版本 | 成熟度 | 权威来源 | 与项目群关系 |
|------|---------|-------|---------|-----------|
| Agentic Web Blueprint | v1.0 | planning baseline | `Agentic_Web/` | 管理长期关系和阶段门 |
| Vlawd / AI Cursor | v2.10 | validating | `AI_cursor_v2/` | L1 Runtime + L7 个人客户端 |
| Capsule Protocol | draft v0.1 | concept | 本模块共享对象草案 | 两个垂直成立后才独立 |

| Campus Lab | concept v0.1 | planned | 未来独立模块 | 知识协作垂直验证 |
| Registry | concept v0.1 | deferred | 未来独立服务 | Capsule 跨环境后启动 |
| Post is Project | concept v0.1 | deferred | 未来独立平台 | Registry 有真实使用后启动 |

版本更新规则：

```text
Vlawd 产品变化：
  更新 AI_cursor_v2，不同步提高 Agentic Web 版本

项目群关系或阶段门变化：
  更新 Agentic_Web，不改变 Vlawd 产品版本

共享 Schema 被两个项目实现：
  创建独立 Capsule Protocol 模块
  在本矩阵记录兼容版本
```

## 权威模块边界

| 内容 | 权威模块 |
|------|---------|
| Vlawd 产品、Runtime、全双工、Task Workspace、Session 和 UI | `AI_cursor_v2/` |
| Agentic Web 长期愿景、层级和项目依赖 | `Agentic_Web/` |
| Capsule / Living Project 跨项目语义草案 | `Agentic_Web/共享架构与协议/` |
| 未来具体协议和服务实现 | 对应独立模块或仓库 |

本注册表只记录 Vlawd 的项目级角色，不重复注册 V2 已经管理的产品功能编号。
