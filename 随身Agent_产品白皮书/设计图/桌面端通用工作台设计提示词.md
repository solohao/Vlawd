# 随身Agent · 桌面端通用工作台设计提示词

## 使用原则

这份提示词不是为了规定颜色、组件类型或组件固定位置，而是为了把产品的**理念、气质和设计目标**讲清楚，让 AI 生图模型在正确方向上充分发挥。

它和 `桌面端设计提示词.md` 的关系是：

- `桌面端设计提示词.md`：偏“销售 / 客户沟通”场景下的桌面端工作台
- 本文件：偏**产品本体**层，强调“通用主场景工作台 + 场景模板可切换”

如果两个文档不一致，以本文件为准。

---

## 产品本体的正确表述

这个产品**不是**：

- 一个 AI 客户工作台
- 一个销售 CRM 助手
- 一个会议纪要软件
- 一个聊天机器人
- 一个工单系统

它的**本体**应该被理解为：

> **一个普通耳机可用、多输入设备可扩展的桌面端 AI 语音工作流系统。**

它的工作方式是：

```text
多输入设备 / 多输入通道
↓
统一语音 / 文本接入
↓
场景识别与理解
↓
专业 Skill / Agent 处理
↓
A2UI 卡片确认
↓
任务 / 工单 / 客户 / 访谈 / 知识等结构化资产
```

因此，桌面端的整体定位是：

> **一个稳定的通用工作台骨架，承接所有沟通输入，并按当前场景动态适配处理结果。**

---

## 通用主场景工作台的核心理念

这个产品的桌面端遵循一个非常重要的原则：

> **界面骨架是稳定的，场景包装是流动的。**

也就是说：

- 用户在产品里看到的导航、布局、视觉语言是**统一**的
- 但**主区域内容会根据当前内容的场景（销售、售后、访谈、会议、创业讨论、个人任务等）动态包装**
- 用户不需要切换“另一个产品”，只需要让系统识别场景或选择场景模板

它和传统垂类 SaaS 的区别：

- 不是“客户工作台”
- 不是“工单系统”
- 不是“访谈工具”
- 而是一个**通用语音工作流工作台**
- 不同场景以**模板 / 视图 / Skill 切换**的方式体现

## 设计气质关键词

- 专业、可信、克制、安静、长期使用不疲劳
- 像知识工作台、推演台、处理台，而不是聊天框
- 智能但不过度炫技，效率工具但不冰冷
- 有一定艺术感、审美节制、空间层次感
- 有现代 AI 产品感，也有严肃工作软件的秩序感
- 隐含“多输入汇聚、AI 理解、人工确认、长期沉淀”的过程感
- 像一个真正在工作的系统，而不是一组装饰性图表

可以理解为：

> **现代专业软件的可靠性 + 高级知识工具的秩序感 + 轻度东方含蓄审美 + AI 时代工作台的未来感。**

---

## 功能意图关键词（不用画出固定组件，只用于引导模型理解系统）

- 多输入设备与多输入通道的接入（耳机、手机、电脑麦、蓝牙麦、录音豆、会议麦、聊天记录、文件、IM 转发）
- 输入收件箱 / 统一处理入口
- AI 转写、场景识别、结构化整理
- 不同场景的模板化呈现（销售、售后、访谈、会议、创业讨论、个人任务等）
- A2UI 卡片：可确认、可编辑、可执行
- 任务、日程、工单、客户、需求、假设、复盘等结构化资产
- 工具同步与连接器（CRM、日历、邮件、知识库、IM）
- Agent / Skill 的运行可视化、步骤进度、待确认动作
- 桌面端主工作台与移动端 / IM 轻入口之间的协同

---

## 应避免的常见误区

- 不要做成普通聊天机器人界面
- 不要做成 CRM 后台
- 不要做成销售运营仪表盘
- 不要做成传统会议纪要软件
- 不要把工单中心、客户中心、销售跟进等当成产品本体
- 不要把整个产品收窄成“客户 AI 工作台”
- 不要堆砌图表、数字、运营 KPI
- 不要让界面看起来像“管理一堆 Agent 的控制台”
- 不要过度赛博朋克 / 过度发光 / 过度未来武器化
- 不要变成手机 App 的放大版

## 通用母提示词

可作为所有桌面端通用工作台设计图的基础提示词：

```text
为一款“桌面端 AI 语音工作流系统”设计高质量产品概念界面。它不是 AI 客户工作台，不是销售 CRM，不是会议纪要软件，而是一个普通耳机可用、多输入设备可扩展的桌面端语音工作台：能够统一接入耳机、麦克风、录音设备、聊天记录、文件和 IM 转发内容，由 AI 进行转写、场景识别和结构化处理，再以可确认、可编辑、可执行的卡片方式呈现给用户，并沉淀为任务、日程、客户、访谈、需求等长期资产。希望界面体现“统一骨架 + 场景模板可切换”的理念：不同场景下主区域内容会自然变化，但导航与产品身份保持稳定。整体气质应专业、可信、克制、长期使用不疲劳，兼具高级设计感和工作台秩序感，不要做成聊天窗口，不要做成 CRM 后台，不要做成 SaaS 运营仪表盘，不要过度炫技。
```


## 补充：普通耳机软件增强层的设计表达

GPT-Realtime 类实时语音模型成熟之后，产品的前台叙事需要更锋利地体现：

> **用户不一定需要购买昂贵的 AI 耳机，也可以通过软件订阅，把已有普通耳机升级成 AI 语音工作入口。**

但这不意味着桌面软件要变成耳机管理器。正确表达应是：

```text
普通耳机增强 = 入口心智
桌面端工作台 = 产品本体
场景模板 = 价值兑现
A2UI / Agent = 能力表现
```

因此，软件界面里可以克制地体现：

- 当前输入设备已连接，例如普通蓝牙耳机、系统麦克风、桌面麦克风
- 实时转写、场景识别、Agent 处理能力已就绪
- 用户可以用已有设备直接开始记录、导入、处理和确认
- 多输入设备只是入口，真正的价值在语音进入工作流之后
- 不应把视觉重点放在某一款耳机硬件或品牌上

## 提示词 8 · 实时语音处理与 AI 运行状态

```text
设计一张“桌面端实时语音处理状态”的概念图。用户正在通过普通耳机或电脑麦克风进行语音输入，系统一边接收声音，一边进行流式转写、场景识别、要点提取、任务候选生成和 Agent 步骤执行。希望界面体现“AI 正在听、正在理解、正在组织成工作流”的过程感，而不是录完以后才开始总结。画面应让用户感到自己的普通输入设备已经被增强为 AI 语音工作入口，同时结果仍然需要通过 A2UI 卡片进行确认、编辑和执行。整体设计应专业、可信、低干扰，避免变成语音波形秀、耳机广告或聊天机器人界面。
```
