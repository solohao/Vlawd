# 01 · 产品定义

## 演进历程

讨论从一连串问题逐步演进：

1. AI 眼镜火了，AI 耳机为什么不火？
2. 能不能做 AI 耳机盒，兼容大部分耳机？
3. 用户已经有耳机盒，不能替代怎么办？
4. 飞书录音豆这类设备做收音是否更合理？
5. 录音豆负责收音，盒子做中枢，耳机播放 AI 回复——这是否成立？
6. viaim 讯飞智能体耳机开始把会议耳机升级为办公 Agent，这是否会形成强竞争？
7. 如果大厂未来做 AI 耳机 + 手机系统联动，我们还能占据什么位置？
8. 是否应先做普通耳机 AI 增强系统，再通过 A2UI、专业 Skill、用户数据和 Agent 市场逐步平台化？

最终收敛到：

> **第一阶段不做 AI 耳机本体，也不先做收音豆硬件，而是先做"普通耳机可用、多输入设备可扩展"的 AI 语音工作流系统：让用户现有耳机成为最低门槛入口，同时兼容手机麦、电脑麦、蓝牙麦、录音豆、会议麦等输入；中长期再通过 A2UI、专业 Skill、用户数据资产和 Agent 市场形成平台。**

---

## 为什么不是先做 AI 耳机？

- 耳机麦克风在耳侧，轻声、近场收音受限
- 耳机 + 充电盒已是成熟生态，难以替代
- 用户不会为 AI 换掉 AirPods / 索尼
- 苹果、华为、三星都在做封闭 AI 耳机
- viaim 已经把 AI 会议耳机升级为办公 Agent，正面硬件竞争成本高

## 为什么不是先做 AI 耳机盒？

- 用户已经有原装耳机盒，多带一个理由不充分
- 无法替代原盒就难以让用户接受
- 只补充 AI 而不解决"收音"核心痛点

## 为什么先做普通耳机 AI 增强系统？

- **故事清楚**：不用买 AI 耳机，让你已有耳机获得 AI 记录、整理、待办和轻执行能力
- **门槛低**：先利用手机/桌面系统已有的音频输入输出能力，不依赖耳机私有协议
- **避开硬件风险**：不在 PMF 未验证前投入供应链、工业设计和库存
- **避开 viaim 正面战场**：不直接卷会议耳机，而从销售、商务、现场售后/实施、访谈、创业等工作流切入
- **为平台蓄水**：持续沉淀用户场景数据、专业 Skill、任务资产和工具连接

普通耳机不是全部收音方案，而是最低门槛入口：

| 场景 | 推荐输入 |
|---|---|
| 电话 / 线上沟通 | 普通耳机、电脑麦克风、系统音频 |
| 会后补记 / 私密输入 | 普通耳机、手机麦克风 |
| 面见客户 / 用户访谈 | 手机桌面收音、蓝牙领夹麦、录音豆 |
| 现场售后 / 实施 | 蓝牙领夹麦、手机麦克风、录音豆 |
| 多人会议 | 会议麦、电脑会议音频、录音豆 |

## 为什么后续仍可能需要收音豆 + 中枢盒？

- **收音豆**:本身就是新设备，自带充电盒理由
- 近场佩戴（衣领、口袋、桌面）收音效果好
- 解决耳机"轻声指令"的硬伤
- 不与用户已有耳机冲突
- **中枢盒**(=收音豆原生充电盒):承担充电 + 存储 + 同步 + 小屏确认 + 本地处理
- 但它们是软件 PMF 之后的增强形态，不是第一阶段的必要前提

## 第一阶段产品定位

### 普通耳机 AI 增强系统

| 维度 | 定位 |
|---|---|
| 入口 | 桌面主工作台 + 轻入口层（全局快捷键、托盘、悬浮状态入口）+ 普通蓝牙耳机、手机麦克风、电脑麦克风、蓝牙领夹麦、录音豆、会议麦、上传音频 |
| 核心场景 | 销售/商务沟通、现场售后/实施、用户访谈、产品讨论、创业想法、会议行动项 |
| 核心能力 | 语音捕捉、轻量触发、状态感知、转写、场景识别、结构化整理、任务生成、轻执行 |
| 差异化 | 不只总结，而是多输入接入 + 专业 Skill + A2UI + 长期资产库 |
| 商业故事 | 不用买 AI 耳机，先让现有耳机可用；需要更好收音时，可接入其他蓝牙/音频设备 |

### 首选用户场景

| 优先级 | 用户 | 为什么需要 |
|---|---|---|
| P0 | 销售 / 商务 / 客户经理 | 通话和拜访频繁，沟通后必须跟进，ROI 直接关联收入 |
| P1 | 现场售后 / 实施 / 客户成功 | 现场问题、客户反馈、工单、回访和知识库沉淀高度依赖语音记录 |
| P2 | 产品经理 / 用户研究 / 创业者 | 访谈、讨论和灵感需要转成需求、假设、验证问题和行动项 |

### 平台能力种子

| 层 | 能力 |
|---|---|
| Input 层 | 普通耳机、手机麦、电脑麦、蓝牙麦、录音豆、会议麦、上传音频 |
| Skill 层 | 销售跟进、现场售后工单、用户访谈、会议行动项、创业假设等专业模板 |
| A2UI 层 | Agent 自动生成可确认、可编辑、可执行的卡片/表单/任务界面 |
| 数据资产层 | 客户、项目、需求、假设、工单、任务、复盘等长期资产 |
| Agent 编排层 | 接入 OpenClaw、ChatGPT、Claude、Kimi、Manus、企业工具 |
| 市场层 | 未来开放官方/第三方/企业私有 Agent 与 Skill |

## 后续硬件增强定位

### 收音豆

| 维度 | 配置 |
|---|---|
| 形态 | 夹子/磁吸/挂绳，约 10g |
| 麦克风 | 双 MEMS 阵列，近场降噪 |
| 输入 | 单击/双击/长按 |
| 反馈 | 状态灯 + 震动（**不发声**） |
| 存储 | 独立录音缓存 |
| 连接 | 蓝牙到中枢盒 / 手机 |

不发声是关键设计决策：

- 保持私密
- 避免社交尴尬
- 避免变成对讲机
- 节省电量与体积

### 中枢盒

| 维度 | 配置 |
|---|---|
| 主要身份 | 收音豆的充电盒升级版 |
| 屏幕 | 小屏：任务卡 / Agent 状态 / 离线队列 |
| 按键 | 物理隐私键、确认键 |
| 计算 | 本地轻量 ASR、关键词检测、任务抽取 |
| 连接 | 蓝牙 + Wi-Fi |
| 存储 | 离线缓存任务和音频片段 |

### 普通耳机（不是自研 AI 耳机）

| 维度 | 配置 |
|---|---|
| 角色 | 语音输入/输出入口 + 私密 AI 回复播放 |
| 协议 | A2DP / HFP / LE Audio |
| 兼容 | 任意品牌蓝牙耳机 |
| 不依赖 | 厂商按键、唤醒词、传感器、AI 协处理器 |

## 与同类产品的差异化

| 维度 | viaim / AI 耳机 | 录音豆 | OpenClaw / 通用 Agent | 我们 |
|---|---|---|---|---|
| 入口 | 自有耳机 | 自有录音硬件 | 软件输入 | **普通耳机 + 手机/桌面** |
| 主场景 | 会议/办公记录 | 录音/转写/总结 | 通用执行 | **垂直语音工作流** |
| 是否换硬件 | 通常需要 | 需要 | 不需要 | **不需要** |
| 输出形态 | 纪要/问答/项目 | 文本/摘要 | 聊天/任务 | **A2UI 卡片 + 工作流动作** |
| 平台路径 | 自有硬件生态 | 录音资产 | Agent 工具链 | **Skill + 数据资产 + Agent 市场** |
| 差异化壁垒 | 语音技术 + 硬件 | 录音体验 | 执行能力 | **专业场景 + 跨生态 + 用户长期资产** |

最大差异：

> **不替代用户耳机，不替代手机，不强迫进入新设备生态，而是先用软件把现有耳机升级为 AI 工作流入口，再用专业 Skill、A2UI 和数据资产逐步平台化。**

## 关于"成为下一个 iPhone"的判断

历史规律：**渐进式补足 > 革命式替代**。

- AirPods 不替代耳机，是配件 → 数千亿市值
- Apple Watch 不替代手机，是补充 → 重要品类
- Kindle 不替代书，是另一种书 → 定义电子书时代
- AI Pin 想替代手机 → 失败
- Rabbit R1 想替代 App → 失败

**我们要做的不是"下一个 iPhone"，而是先做"普通耳机的 AI 工作流升级层"**——用一个清晰垂类应用切入，再长成跨耳机、跨设备、跨工具的语音 Agent 工作流平台。

## 一句话总结

**不是 AI 耳机，不是录音豆，不是 OpenClaw 套壳，而是先从普通耳机 AI 增强切入、逐步兼容多输入设备的语音工作流平台：用耳机、手机麦、电脑麦、蓝牙麦、录音豆等入口捕捉语音，用专业 Skill 和 A2UI 把内容变成可信、可控、可执行、可沉淀的工作资产，并逐步开放 Agent 市场。**


## 补充：第一阶段更适合桌面端工作台

第一阶段更准确的产品本体应收束为“桌面端 AI 客户沟通工作台”，而不是同时把手机 App、小程序和硬件都作为首发重点。

原因包括：

- 桌面端更适合承载多输入导入、转写、A2UI 确认、长任务处理与数据资产沉淀
- 个人创业者在桌面端更容易避开双端适配、后台权限、支付与审核等额外成本
- 手机端更适合承担轻入口、补充录音、结果查看和消息触达，而不是第一阶段完整产品本体
- 微信、QQ、飞书、钉钉、QClaw 等入口可以在中期承担远程触达与“变相移动端”能力

因此第一阶段更准确的结构是：

```text
桌面端工作台
+ 多输入导入
+ 官方高价值 Skill
+ A2UI 卡片确认
+ 轻量工具同步
```

## 补充：后续硬件形态进一步收束

中后期的硬件增强不应简单理解为“中枢盒”，而应进一步分化为两类：

### 1. Agent Slate

面向个人和专业用户的低干扰控制板，用于：

- 输入设备管理
- A2UI 轻确认
- 当前场景状态查看
- 现场低摩擦触发下一步动作

### 2. 企业中枢工作站

面向公司内部部署的组织级 Agent 基础设施，用于：

- 统一设备接入
- 统一权限与审计
- 统一企业系统同步
- 私有化工作流与数据边界控制

这样可以避免把后期硬件误解为“另一个耳机盒”，而更准确地理解为：个人控制板与企业基础设施两条演进路径。
