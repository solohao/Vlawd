# 06 · 演进路线图

## 总体节奏

```text
2026：普通耳机 AI 增强 MVP，验证垂类 PMF
2027：专业 Skill + A2UI，沉淀用户数据资产
2028：Agent / Skill 市场雏形，扩展相邻场景
2029：硬件增强与企业版，完成更强闭环
2030：建立语音工作流生态护城河，对抗大厂
```

**窗口期 2026-2028 是大厂耳机 + 手机 AI 完全闭环前，抢占普通耳机 AI 工作流层的关键阶段。**

---

## 阶段 1 · 2026：普通耳机 AI 增强 MVP

### 目标

- 验证 PMF
- 攒第一批用户
- 不做硬件
- 找到一个高价值垂类场景

### 形态

```text
普通蓝牙耳机 + 手机/桌面 App + 官方 Skill + 轻执行
```

### 工作内容

- 普通耳机 / 手机 / 桌面语音捕捉
- 销售/商务或产品访谈 Skill
- 语音转写 + 场景识别 + 结构化整理
- A2UI 卡片雏形（客户卡、任务卡、访谈卡、假设卡）
- 基础轻执行（邮件草稿、日历、待办、CRM/飞书/Notion 同步）

### 验证问题

- "不用买 AI 耳机，让普通耳机变 AI 工作流入口"的故事是否成立？
- 销售/商务/访谈用户是否愿意持续使用？
- 用户是否愿意付费？
- 专业 Skill 是否比通用总结有明显价值？

---

## 阶段 2 · 2027：专业 Skill + A2UI + 数据资产

### 目标

- 从单点工具升级为垂直工作流系统
- 形成 3-5 个官方高质量 Skill
- 建立客户/项目/需求/假设/任务等资产库

### 形态

```text
+ 官方 Skill 系统
+ A2UI 卡片/表单/任务队列
+ 用户长期数据资产库
+ 工具连接器（CRM / 飞书 / Notion / 日历）
+ 可选接入 Plaud / 飞书录音豆 / 领夹麦
```

### 工作内容

- 销售跟进、用户访谈、产品讨论、会议行动项、创业假设 Skill
- A2UI 通用组件抽象
- 个人/团队资产库
- 用户自定义字段和模板
- 可选集成现成录音硬件验证轻声输入价值

### 验证问题

- 垂直 Skill 是否能产生明显 ROI？
- 用户是否持续沉淀业务资产？
- A2UI 是否显著降低 Agent 使用门槛？
- 现成录音硬件是否值得深度合作？

---

## 阶段 3 · 2028：Agent / Skill 市场雏形

### 目标

- 从官方 Skill 扩展到专家/第三方 Skill
- 开放有限的 Skill 创建和分发能力
- 扩展到相邻场景

### 形态

```text
+ Skill Builder
+ Agent / Skill 市场内测
+ A2UI 模板市场
+ OpenClaw / 多 Agent 后端接入
+ 企业团队版
+ 可选硬件合作版
```

### 产品分层

| 版本 | 配置 | 价格 |
|---|---|---|
| Pro 版 | 专业 Skill + A2UI + 工具同步 | 订阅 |
| Team 版 | 团队资产库 + 权限 + CRM/飞书集成 | 按席位 |
| Market 内测 | 专家 Skill / 企业私有 Skill | 抽成 / 定制 |

### 工作内容

- Skill 创建器
- A2UI 模板规范
- Agent 权限与审核
- 专家方法论合作
- 企业私有 Skill

### 验证问题

- 第三方专家是否愿意发布 Skill？
- 用户是否愿意购买专业 Agent / Skill？
- 平台审核、权限和安全机制是否可控？

---

## 阶段 4 · 2029：硬件增强 + 企业闭环

### 目标

- 在软件 PMF 和平台雏形成立后，再推出硬件增强
- 服务专业版、企业版、极客版
- 加强离线、隐私、本地处理和低摩擦输入

### 形态

```text
+ 自研或合作收音豆
+ 中枢盒 / 私密确认设备
+ 端侧 ASR / 轻量 LLM
+ 企业私有部署版
+ 可选 AI 工作站 / 本地模型主机
```

### 产品分层

| 版本 | 配置 | 价格 |
|---|---|---|
| 专业硬件版 | 收音豆 / 领夹麦 + 软件 Pro | 硬件 + 订阅 |
| 旗舰版 | 收音豆 + 中枢盒 / 工作站 + 本地模型 | ¥3000-15000 |
| 企业版 | 私有部署 + 席位订阅 | 按席位 |

### 验证问题

- 硬件是否显著提升输入频率和留存？
- 企业是否愿意为隐私、本地处理和私有 Skill 付费？
- 平台收入是否超过硬件一次性收入？

---

## 阶段 5 · 2030+：平台 + 生态

### 目标

- 生态飞轮启动
- 平台收入 > 硬件收入
- 建立网络效应
- 对抗大厂通用 AI 耳机能力

### 形态

```text
+ 完整开发者生态
+ Agent / Skill 市场成熟
+ A2UI 模板市场成熟
+ 联名硬件 / 行业硬件合作
+ 企业 API 业务规模化
+ 多语言 / 多地区
```

### 风险

- 大厂入局（Apple / Google / 华为 / 小米 / 飞书 / viaim）
- 必须靠垂直场景深度、用户数据资产、A2UI 和 Agent / Skill 生态抵抗

## 核心约束

不要犯的错误：

1. **不要一开始做硬件**：先用现成的验证
2. **不要绑死 OpenClaw**：保持 Agent 后端可替换
3. **不要早期开放平台**：用户少、Skill 少、数据资产少时生态做不起来
4. **不要只做通用会议纪要**：viaim、飞书、钉钉、腾讯会议会快速挤压
5. **不要全押本地大模型**：端侧 LLM 性价比要持续评估
6. **不要追求一步到位的"AI 工作站"**：架构预留，但产品分阶段推出
7. **不要忽视合规**：录音法规和 AI 责任要早做

---

## 一句话

**不要赌"AI 必然达到完美"，而要赌"普通耳机语音入口会被 AI 重新组织，且垂直工作流会先于通用 Agent 跑出 PMF"。最稳的策略是：2026 先做普通耳机 AI 增强 MVP，跑通一个高价值垂类 Skill；2027-2028 沉淀 A2UI、数据资产和 Agent / Skill 市场雏形；2029 以后再用硬件和企业能力增强平台护城河。**


## 补充：桌面端优先与分场景产品线

路线图在产品形态上需要进一步明确：

### 2026：桌面端优先的垂类 MVP

```text
桌面端 AI 客户沟通工作台
+ 多输入导入
+ 官方高价值 Skill
+ A2UI 卡片雏形
+ 轻量工具同步
```

重点不是完整手机 App，而是先让桌面端成为主工作台。

### 2027：数据资产与 IM 远程桥接

```text
桌面端工作台
+ 微信 / QQ / 飞书 / 钉钉 / QClaw 轻入口
+ 个人 / 团队资产库
+ 更多官方 Skill
```

### 2028：团队版与企业 Agent Hub 软件版

```text
Team 版
+ 权限与共享资产
+ 企业私有 Skill
+ 企业 Agent Hub 软件部署版
+ 有限的 Skill Builder
```

### 2029：硬件增强与企业中枢工作站

```text
Agent Slate
+ 收音豆 / 输入增强设备
+ 企业中枢工作站硬件版
+ 本地模型 / 端侧能力增强
```

同时，核心优先场景可收束为：

- P0：销售 / 商务 / 客户经理
- P1：现场售后 / 实施 / 客户成功
- P2：产品经理 / 用户研究 / 创业者

其中桌面端是这些场景共同的主工作台，区别主要体现在 Skill、输入设备选择和输出工作流配置上。
