# 随身 AI Agent 系统 · 总览

> 本系列文档整理自一段长时间的产品策略讨论，记录我们对一个"普通耳机 AI 增强切入 → 多输入设备接入 → 专业 Skill → A2UI → Agent 市场"的随身 Agent 平台路线的思考全过程。

---

## 一句话定义

**一个先从普通耳机 AI 增强切入、逐步扩展到多输入设备的随身 Agent 平台：让用户通过耳机、手机、电脑麦、蓝牙麦、录音豆等入口，把语音沟通、想法和任务转化为可整理、可确认、可执行、可沉淀的工作流资产。**

它不是 AI 耳机硬件，不是录音笔，也不是单纯的手机 App，而是：

> **Agent 时代的多输入语音工作流入口层。**

---

## 产品全景

### 第一阶段：普通耳机 AI 增强系统

- **现有耳机**
  - 作为用户已有的语音输入/输出入口
  - 不要求更换 AirPods、华为、小米、索尼等现有设备
  - 不依赖厂商私有按键、唤醒词、传感器和 AI 协处理器

- **手机 / 桌面 App**
  - 语音捕捉、转写、结构化整理
  - 任务卡、客户卡、项目卡、假设卡
  - 权限确认、轻执行、工具同步

- **多输入设备接入**
  - 普通耳机负责电话、语音补记和私密回复
  - 手机麦克风、电脑麦克风负责桌面和线上场景
  - 蓝牙领夹麦、录音豆、会议麦负责面访、售后、实施和多人场景

- **专业 Skill**
  - 销售跟进、用户访谈、产品讨论、创业假设、会议整理
  - 把语音转成垂直场景的结构化结果和下一步动作
  - 先由官方提供，后续开放专家和第三方创建

- **A2UI 交互层**
  - Agent 不只输出文本，而是生成可确认、可编辑、可执行的卡片界面
  - 让用户能修改字段、确认动作、同步工具、追踪结果

- **Agent / Skill 市场**
  - 基于用户长期数据和场景模板，逐步生成个人/团队 Agent
  - 未来开放官方 Agent、行业 Skill、企业私有 Agent 和工具连接器

### 后续硬件增强

- **收音豆**
  - 作为轻声、高质量、随身语音捕捉的增强外设
  - 在软件 PMF 验证后再自研或合作，不作为第一阶段前提

- **中枢盒 / 工作站**
  - 作为离线缓存、本地模型、隐私计算和 Agent 状态确认的进阶形态
  - 面向专业版、企业版、极客版逐步推出

- **OpenClaw / 多 Agent 后端**
  - 不是默认绑定的唯一引擎，而是高级任务执行层之一
  - 与 ChatGPT、Claude、Kimi、Manus、企业 RPA 等后端并列接入

## 解决的核心问题

- **用户不想换 AI 耳机** → 先让现有普通耳机获得 AI 工作流能力，再按场景接入更合适的收音设备
- **普通耳机面对面收音不足** → 用手机麦、蓝牙麦、录音豆、会议麦补足高价值场景闭环
- **大厂会占通用入口** → 避开通用助手，沉淀垂直场景 Skill 和用户数据资产
- **会议纪要容易被 viaim 占据** → 不正面卷会议耳机，优先切销售/商务/访谈/创业等语音工作流
- **OpenClaw 只是执行引擎** → 我们做语音输入后的场景理解、A2UI 确认、任务资产和 Skill 编排
- **Agent 执行不可信** → 用可视化卡片、权限确认、预算限制和任务状态追踪降低风险
- **平台冷启动难** → 先做一个垂类应用，底层按平台能力设计

## 四大机制

### 1. 普通耳机 AI 化

```text
普通耳机 / 手机 / 桌面麦克风
↓
语音捕捉
↓
AI 转写与场景识别
↓
专业 Skill 处理
```

不从硬件替代开始，而从存量耳机的软件增强开始。

### 2. Skill 化工作流

```text
语音输入 → 场景判断 → Skill 模板 → 结构化结果 → 下一步动作
```

先由官方提供高质量 Skill，再逐步开放用户自定义和第三方发布。

### 3. A2UI 确认层

```text
Agent 输出
↓
卡片 / 表单 / 时间线 / 任务队列
↓
用户确认 / 修改 / 同步 / 执行 / 复盘
```

Agent 不能只会聊天，必须把结果变成用户可操作的界面。

### 4. 数据资产飞轮

- **输入越多**：客户、项目、需求、假设、任务、复盘越完整
- **资产越多**：系统越懂用户场景和偏好
- **Skill 越准**：专业输出越贴合真实工作流
- **Agent 越个性化**：最终能自动生成个人/团队 Agent

## 核心战略判断（速读版）

1. **方向对**：普通耳机和蓝牙音频设备已经普及，但其上的 AI 工作流层尚未定型，这是平台窗口期。
2. **第一阶段要清楚**：不是先造硬件，而是先做"普通耳机可用、多输入设备可扩展"的软件产品。
3. **不能只做通用助手**：大厂会吃掉通用能力，必须沉淀销售、访谈、创业等专业 Skill。
4. **平台从垂类应用长出来**：先做一个高价值垂类应用，底层沉淀 A2UI、Skill、数据资产和 Agent 编排。
5. **窗口期 2026-2028**：在大厂耳机 + 手机 AI 完全闭环前，抢占跨生态、跨工具的专业工作流层。


## 补充：桌面端优先与多形态演进

当前路线需要进一步收束为：

- 第一阶段以桌面端作为主工作台，负责语音导入、转写、结构化整理、A2UI 确认和资产沉淀
- 手机不强行承担完整产品本体，而是作为轻入口、补充录音入口和消息触达端
- 中期通过微信、QQ、飞书、钉钉、QClaw 等入口形成“变相移动端”体验
- 中后期再分化出 Agent Slate 与企业中枢工作站两种硬件 / 基础设施形态

可以把整体结构理解为：

```text
桌面端 = 主工作台
手机 / IM = 轻入口与远程触达
Agent Slate = 个人 / 专业用户的轻控制板
企业中枢工作站 = 组织级 Agent 基础设施
```
