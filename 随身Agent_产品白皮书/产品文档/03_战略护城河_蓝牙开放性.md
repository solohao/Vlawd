# 03 · 战略护城河 · 蓝牙开放性

## 核心洞察

> **借助蓝牙耳机和各类蓝牙/音频输入设备这个已经普及的入口，绕过 AI 眼镜和 AI 耳机硬件生态的封闭竞争，先建立一个多输入设备之上的 AI 工作流层。**

这是结构性的窗口，但不是硬件控制权优势，而是应用层和工作流层的窗口。

苹果、华为、小米可以让自家耳机 + 手机的 AI 体验越来越强，但**无法消灭跨品牌耳机、跨系统设备、跨办公软件、跨收音设备混用的现实场景**。只要用户仍然通过普通耳机、手机、电脑麦、蓝牙麦、录音豆、会议麦等设备产生语音输入，就存在软件工作流层的机会。

---

## 平台型产品的核心规律

```text
基础协议/技术 → 已经统一标准化
↓
硬件/带宽/成本 → 已经成熟廉价
↓
但应用层 → 还没人真正做出来
↓
窗口期出现 → 早期进入者占据平台位置
```

**真正的平台型公司，几乎都诞生在这个窗口里。**

## 历史案例验证

| 标准化基础 | 窗口期 | 应用层赢家 |
|---|---|---|
| HTTP + 万维网 | 1995-2005 | 谷歌、亚马逊、Facebook |
| iOS / Android | 2009-2015 | Uber、Instagram、抖音 |
| 4G + 智能手机 | 2015-2020 | 抖音、快手、TikTok |
| USB / Type-C | 2000-2025 | 各类外设、扩展坞、eGPU |
| GPS 开放 | 2000-2015 | 谷歌地图、Uber、滴滴 |
| TCP/IP | 1985-2000 | 思科、雅虎、整个互联网 |
| HTML5 + WebKit | 2010-2020 | Slack、Notion、Figma、Zoom |
| 开源大模型 + API | 2023-2028 | Cursor、Perplexity、Character.AI |

**我们正处于"多输入音频设备 + AI 工作流"窗口期（2026-2028）**。

窗口期的本质不是能控制所有耳机，而是：

```text
普通耳机、手机麦、电脑麦、蓝牙麦、录音豆等音频入口已经普及
↓
用户已经习惯用不同设备通话、会议、录音、补记
↓
但语音进入系统之后
↓
由谁负责理解、整理、确认、执行、沉淀
↓
尚未形成标准答案
```

## 蓝牙耳机 vs AI 眼镜 对比

| 维度 | 蓝牙耳机 | AI 眼镜 |
|---|---|---|
| 标准化协议 | ✅ A2DP/HFP/LE Audio | ❌ 无统一标准 |
| 跨厂商兼容 | ✅ 任意品牌通用 | ❌ 各家独立生态 |
| 基础音频接入 | ✅ 标准系统能力 | ❌ 必须用厂商 SDK |
| 厂商能封锁 | ❌ 不能 | ✅ 可以 |
| 创业者机会 | ✅ 软件工作流层 | ❌ 单一生态层 |
| 我们的产品适配 | ✅ 基础输入输出可复用 | ❌ 每家单独开发 |

## 蓝牙为什么能统一？

**统一不是"善意结果"，而是被多重压力逼出来的：**

1. **前一代标准失败的教训**（IrDA / HomeRF）
2. **单一厂商无法主导市场**（90 年代末没绝对霸主）
3. **硬件成本的强烈压力**（重复研发不可承受）
4. **网络效应陷阱**（不统一就没人买）
5. **监管和行业标准压力**（IEEE 802.15.1）

## AI 眼镜为什么不统一？

1. **每家都觉得"自己能赢"**（Meta / Apple / Google / 国内厂商）
2. **还没有"前一代教训"**（AR 还是第一代）
3. **存在垄断雄心**（都想做下一个 iPhone）
4. **技术差异化的天然存在**（显示方案各异）
5. **用户规模太小，痛点不够**（年出货还在百万-千万级）

按蓝牙的历史规律，AI 眼镜要统一需要：

- 第一代私有协议失败（3-5 年）
- 没有绝对霸主出现
- 用户规模到 1-2 亿台（5-8 年）
- 监管或行业组织推动（欧盟可能先动手）

**综合预测**：2026-2028 各家割据，2028-2030 安卓阵营可能内部统一，2030-2035 才可能出现真正的跨厂商标准。

## 蓝牙的 5 层开放程度

| 层 | 开放度 | 我们的策略 |
|---|---|---|
| 蓝牙音频播放 A2DP | ✅ 完全开放 | 核心依赖 |
| 蓝牙通话 HFP/LE Audio | ✅ 基本开放 | 输入输出入口 |
| 耳机按键/手势 | ⚠️ 部分封闭 | 不作为第一阶段核心承诺 |
| 自定义唤醒词 | ❌ 几乎不开放 | 不依赖，用 App / 快捷入口 / 后续收音豆 |
| 传感器 / AI 协处理器 | ❌ 完全封闭 | 不依赖 |

## 关键策略

### 1. 把"耳机依赖"降到最低

耳机只承担：
- ✅ 播放 AI 回复（A2DP）
- ✅ 通话音频（HFP / LE Audio）
- ✅ 作为系统音频输入设备之一

不依赖：
- ❌ 自定义按键
- ❌ 自定义唤醒词
- ❌ 传感器数据
- ❌ AI 协处理器

### 2. 把核心价值放在软件工作流层

- **场景识别** → 判断语音属于销售、访谈、会议、创业假设还是个人任务
- **专业 Skill** → 把语音变成垂直场景结构化结果
- **A2UI** → 把 Agent 输出变成可确认、可编辑、可执行界面
- **数据资产** → 沉淀客户、项目、需求、假设、任务和复盘

### 3. 用通用入口做"软兼容"

- 手机 App 录音 / 桌面快捷键 / 系统分享 / 上传音频 / 通话后补记
- 普通耳机只作为用户已有输入输出设备，不承诺深度接管
- 后续再用快捷指令、通知栏、小组件、媒体按键做低摩擦增强

### 4. 分级支持不同体验

- ✅ **基础可用**：任意普通耳机 + 手机/桌面 App
- 🟡 **增强体验**：快捷入口、通话后补记、桌面端、工具同步
- 💡 **最佳体验**：软件 PMF 后接入收音豆、中枢盒或厂商合作硬件

## 真正的竞争对手不是 ElevenLabs

| 类型 | 代表 | 是否对手 |
|---|---|---|
| 语音基础设施 | ElevenLabs / Deepgram / Whisper | ❌ 应该是供应商 |
| Agent 引擎 | OpenClaw / Claude / Manus | ❌ 应该是后端 |
| 大厂垂直整合 | Apple Intelligence + AirPods | ⚠️ 真竞争对手 |
| 大厂垂直整合 | Google Gemini + Pixel Buds | ⚠️ 真竞争对手 |
| AI 智能体耳机 | viaim / 讯飞智能体耳机 | ⚠️ 办公记录与项目 Agent 强竞品 |
| 企业生态 | 飞书 / 钉钉 / 企业微信 | ⚠️ 在企业场景有优势 |
| 录音豆品牌 | Plaud / Anker | ⚠️ 可能向 Agent 演进 |
| 新硬件失败者 | AI Pin / Rabbit R1 / Humane | ❌ 已证伪 |

**护城河不是语音技术，也不是耳机协议控制权，而是专业 Skill + A2UI + 用户长期数据资产 + 跨工具 Agent 编排形成的端到端工作流体验。**

---

## 一句话

**AI 眼镜虽然热，但被各家锁死；AI 耳机未来也会被手机厂商垂直整合。真正的窗口不在"控制耳机"，而在"普通耳机语音输入之后的 AI 工作流层"：谁先占据专业场景、A2UI、Skill 和用户数据资产，谁就有机会从垂类应用长成平台。**


## 补充：蓝牙边界修正

这里需要进一步明确：我们的机会来自“蓝牙音频设备已经被操作系统标准化为通用输入 / 输出能力”，而不是来自对耳机底层协议的直接控制。

因此第一阶段的真实边界应表述为：

- 我们接入的是系统暴露出来的音频输入源或用户主动导入的音频 / 文本内容
- 不承诺直接接管所有蓝牙耳机的底层连接与私有能力
- 不把厂商私有按键、唤醒词、传感器和协议控制权作为产品前提
- 护城河仍然在专业 Skill、A2UI、长期数据资产和工作流编排层

更准确地说：蓝牙耳机之所以比 AI 眼镜更适合作为早期入口，不是因为我们能直接控制所有耳机，而是因为它们的基础音频能力已被系统标准化，能低成本接入桌面端和软件工作流。
